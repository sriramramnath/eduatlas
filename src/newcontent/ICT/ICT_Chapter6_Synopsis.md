## Quick Reference Summary

### 1. Types of Malware

| Type | Key Characteristic | Spreads By |
|------|-------------------|------------|
| **Virus** | Attaches to host files | Infected files, email attachments |
| **Worm** | Self-replicating, no host needed | Network vulnerabilities |
| **Trojan** | Disguised as legitimate software | User downloads |
| **Ransomware** | Encrypts files, demands payment | Email, malicious downloads |
| **Spyware** | Secretly monitors activity | Bundled software, downloads |
| **Adware** | Displays unwanted ads | Free software bundles |

---

### 2. Phishing Types

| Type | Target | Method |
|------|--------|--------|
| **Email Phishing** | Mass audience | Fake emails from organisations |
| **Spear Phishing** | Specific individuals | Personalised fake emails |
| **Whaling** | Senior executives | Targeted high-value attacks |
| **Smishing** | Mobile users | Phishing via SMS |
| **Vishing** | Phone users | Voice phishing calls |

---

### 3. Phishing Warning Signs

```
 SUSPICIOUS EMAIL INDICATORS:
• Sender address doesn't match organisation
• Generic greeting ("Dear Customer")
• Urgent action required
• Suspicious links (check before clicking)
• Spelling/grammar errors
• Requests for personal information
• Threats (account closure, legal action)
```

---

### 4. Data Interception Methods

| Method | How It Works | Prevention |
|--------|--------------|------------|
| **MITM Attack** | Attacker intercepts communication | Use HTTPS, VPN |
| **Packet Sniffing** | Captures network data packets | Encrypt transmissions |
| **Eavesdropping** | Listens to unencrypted traffic | Secure Wi-Fi, encryption |

---

### 5. Unauthorised Access Methods

| Attack Type | Method | Prevention |
|-------------|--------|------------|
| **Brute Force** | Tries all password combinations | Strong passwords, account lockout |
| **Dictionary Attack** | Uses common password lists | Unusual passwords |
| **Credential Stuffing** | Uses leaked credentials | Unique passwords per account |
| **Social Engineering** | Manipulates people | User education |

---

### 6. Protection Measures

#### Technical Measures

| Measure | Function |
|---------|----------|
| **Firewall** | Filters network traffic |
| **Antivirus** | Detects/removes malware |
| **Encryption** | Protects data confidentiality |
| **Strong Passwords** | Prevents unauthorised access |
| **2FA/MFA** | Additional authentication layer |
| **Software Updates** | Patches security vulnerabilities |

#### Organisational Measures

| Measure | Purpose |
|---------|---------|
| **Access Control** | Limits access to authorised users |
| **AUP** | Defines acceptable IT usage |
| **Backup Policy** | Ensures data recovery |
| **User Training** | Educates about threats |

---

### 7. Password Best Practices

**Strong Password:**
```
✓ At least 12 characters
✓ Mix upper/lower case letters
✓ Include numbers and symbols
✓ Not based on personal info
✓ Unique for each account
✓ Changed regularly
```

**Authentication Hierarchy:**
- Single-factor: Password only (least secure)
- Two-factor (2FA): Password + code
- Multi-factor (MFA): Password + app + biometric (most secure)

---

### 8. Backup Strategies

| Strategy | What's Backed Up | Speed | Storage |
|----------|-----------------|-------|---------|
| **Full** | All data | Slowest | Largest |
| **Incremental** | Changed since last backup | Fastest | Smallest |
| **Differential** | Changed since last full | Medium | Medium |

**3-2-1 Backup Rule:**
- 3 copies of data
- 2 different storage types
- 1 copy offsite

### 9. Consequences of Data Breaches

| Level | Consequences |
|-------|--------------|
| **Personal** | Identity theft, financial loss, privacy violation |
| **Organisational** | Financial costs, reputational damage, legal action, fines |

---

### 10. High-Yield Exam Points

**Must-Know Facts:**

- **Virus needs host file** — worm doesn't
- **Trojan disguises as legitimate** — tricks users
- **Ransomware encrypts files** — demands payment
- **Phishing steals credentials** — fake emails/sites
- **HTTPS encrypts web traffic** — look for padlock
- **2FA adds security layer** — password + something else
- **Updates patch vulnerabilities** — always update software
- **Backups enable recovery** — 3-2-1 rule

**Security Measures Hierarchy:**

```
Prevention → Detection → Response
    ↓             ↓           ↓
 Firewall    Antivirus    Backup
 Encryption   Monitoring   Recovery plan
 Strong PW    Alerts       Incident response
```

### 11. Quick Definition List

| Term | Definition |
|------|------------|
| **Malware** | Malicious software harmful to systems |
| **Phishing** | Fraudulent attempt to steal information |
| **Ransomware** | Malware encrypting files for ransom |
| **Firewall** | Device filtering network traffic |
| **Encryption** | Converting data to secure unreadable form |
| **Authentication** | Verifying identity for access |
| **2FA/MFA** | Multiple authentication factors |
| **AUP** | Policy defining acceptable IT use |
| **Social Engineering** | Manipulating people for information |
| **MITM** | Man-in-the-middle attack intercepting data |

---

### 12. Exam Tips

1. **Know malware differences** — virus vs worm vs trojan
2. **Identify phishing signs** — urgency, generic greetings, suspicious links
3. **Explain layered security** — multiple measures together
4. **Mention both technical AND human measures**
5. **Use specific examples** — not just "security software"
6. **Connect measures to threats** — why each measure helps
