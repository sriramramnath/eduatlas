## Quick Reference Summary

### 1. Cloud Service Types

| Type | Full Name | Provides | Examples |
|------|-----------|----------|----------|
| **IaaS** | Infrastructure as a Service | Virtual machines, storage, networking | AWS EC2, Azure VM |
| **PaaS** | Platform as a Service | Development tools, databases | Heroku, Google App Engine |
| **SaaS** | Software as a Service | Complete applications | Google Docs, Microsoft 365 |

---

### 2. Cloud Deployment Models

| Model | Description | Key Characteristic |
|-------|-------------|-------------------|
| **Public Cloud** | Shared infrastructure | Multiple organisations share resources |
| **Private Cloud** | Dedicated infrastructure | Single organisation has exclusive use |
| **Hybrid Cloud** | Combination | Mix of public and private resources |

---

### 3. Key Cloud Characteristics

```
CLOUD ESSENTIALS:
✓ On-demand self-service
✓ Broad network access
✓ Resource pooling
✓ Rapid elasticity
✓ Measured service (pay-as-you-go)
```

### 4. Cloud Service Comparison

| Aspect | IaaS | PaaS | SaaS |
|--------|------|------|------|
| **User manages** | Applications, data, runtime | Applications, data | Data only |
| **Provider manages** | Infrastructure | Platform + infrastructure | Everything |
| **Flexibility** | Highest | Medium | Lowest |
| **Ease of use** | Requires expertise | Development skills needed | Just use it |
| **Control** | Most control | Medium control | Least control |

### 9. High-Yield Exam Points

**Must-Know Facts:**

- **IaaS = infrastructure** — servers, storage, networking
- **PaaS = platform** — development tools, databases
- **SaaS = software** — complete applications (most common)
- **Public cloud = shared** — lower cost, less control
- **Private cloud = dedicated** — more control, higher cost
- **Hybrid = combination** — best of both worlds
- **Cloud needs internet** — always mention this limitation
- **Pay-as-you-go model** — only pay for what you use

**Key Distinctions:**

| Question | Answer |
|----------|--------|
| What is Gmail? | SaaS |
| What is AWS EC2? | IaaS |
| What is Heroku? | PaaS |
| What is shared cloud? | Public cloud |
| What is dedicated cloud? | Private cloud |

### 11. Exam Tips

1. **Know the three service types** — IaaS, PaaS, SaaS with examples
2. **Distinguish deployment models** — public, private, hybrid
3. **Balance benefits and risks** — always consider both
4. **Mention internet dependency** — critical limitation
5. **Use specific examples** — name real services
6. **Consider security** — both features and risks
