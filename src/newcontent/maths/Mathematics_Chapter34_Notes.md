# 1. Statistical Investigation

## 1.1 The Statistical Enquiry Cycle

The statistical enquiry cycle (also called the data handling cycle) is a systematic approach to investigating questions using data. It consists of **four main stages**:

```
┌─────────────────────────────────────────────────────────┐
│                    STATISTICAL CYCLE                     │
│                                                          │
│    ┌──────────┐                                         │
│    │  PLAN    │ ──────► Formulate hypothesis/questions   │
│    └────┬─────┘         Decide what data to collect      │
│         │               Choose collection method         │
│         ▼                                                │
│    ┌──────────┐                                         │
│    │ COLLECT  │ ──────► Design data collection sheets    │
│    └────┬─────┘         Carry out survey/experiment      │
│         │               Record data systematically       │
│         ▼                                                │
│    ┌──────────┐                                         │
│    │ PROCESS  │ ──────► Organise data in tables          │
│    └────┬─────┘         Calculate statistics             │
│         │               Create diagrams                  │
│         ▼                                                │
│    ┌──────────┐                                         │
│    │ DISCUSS  │ ──────► Interpret results                │
│    └──────────┘         Draw conclusions                 │
│                         Evaluate limitations             │
└─────────────────────────────────────────────────────────┘
```

## 1.2 Types of Data

### Primary vs Secondary Data

| **Primary Data** | **Secondary Data** |
|------------------|-------------------|
| Collected **firsthand** by the researcher | Data already collected by **someone else** |
| For a **specific purpose** | Used for a **different purpose** than originally intended |
| **Examples:** surveys, experiments, questionnaires | **Examples:** internet, government statistics, textbooks |
| ✓ More reliable and relevant | ✓ Quicker and cheaper to obtain |
| ✗ Time-consuming and expensive | ✗ May be outdated or unreliable |

### Qualitative vs Quantitative Data

| **Qualitative Data** | **Quantitative Data** |
|---------------------|----------------------|
| **Descriptive** data (words, categories) | **Numerical** data (numbers, measurements) |
| Cannot be measured numerically | Can be counted or measured |
| **Examples:** eye colour, favourite subject, opinions | **Examples:** height, weight, test scores, age |

>  **Common Mistake:** Don't confuse data type with data collection method. A questionnaire can collect both qualitative and quantitative data.

### Discrete vs Continuous Data (Sub-types of Quantitative)

| **Discrete Data** | **Continuous Data** |
|-------------------|---------------------|
| Can only take **specific values** | Can take **any value** within a range |
| Counted in **whole numbers** | Measured on a **scale** |
| **Examples:** number of children, shoe size, goals scored | **Examples:** height, weight, time, temperature |

## 1.3 Data Collection Methods

### Questionnaires and Surveys

**Designing effective questions:**

✓ **Good questions:**
- Clear and unambiguous
- Not leading or biased
- Appropriate response options
- Relevant to the investigation

✗ **Bad questions:**
- "Don't you think homework is too hard?" (Leading)
- "How much TV do you watch?" (Ambiguous - hours? minutes? per day/week?)
- Double-barrelled questions asking two things at once

**Response options:**
- **Closed questions:** Fixed response options (tick boxes)
  - Easier to analyse
  - Limited detail
  
- **Open questions:** Free text response
  - More detailed information
  - Harder to analyse and categorise

**Example:**

❌ **Poor question:** "Do you agree that school uniform should be banned because it's expensive?"

✓ **Better question:** "What is your opinion on school uniform?"
  - Strongly agree with keeping it
  - Slightly agree with keeping it
  - Neutral
  - Slightly disagree with keeping it
  - Strongly disagree with keeping it

### Sampling Methods

**Why sample?**
- Impossible or impractical to survey entire population
- Cheaper and quicker
- If done well, gives accurate representation

#### Random Sampling

> **Definition:** Every member of the population has an **equal chance** of being selected.

**Methods:**
1. **Simple random sampling:** Assign numbers, use random number generator
2. **Systematic sampling:** Select every $n^{th}$ person from a list
3. **Stratified sampling:** Proportional representation from each subgroup

**Stratified Sampling Formula:**

$$\text{Number from each stratum} = \frac{\text{size of stratum}}{\text{total population}} \times \text{sample size}$$

**Worked Example:**

A school has 1200 students: 480 boys and 720 girls. A stratified sample of 100 students is needed.

**Solution:**

*Number of boys:*
$$\frac{480}{1200} \times 100 = 40 \text{ boys}$$

*Number of girls:*
$$\frac{720}{1200} \times 100 = 60 \text{ girls}$$

> **Key Insight:** The sample proportions match the population proportions: 40% boys and 60% girls.

#### Non-Random Sampling

| **Method** | **Description** | **Advantages** | **Disadvantages** |
|------------|-----------------|----------------|-------------------|
| **Quota sampling** | Interview until quota from each group is filled | Quick, ensures representation | Non-random, interviewer bias possible |
| **Convenience sampling** | Use whoever is easily available | Very quick, cheap | Highly biased, unrepresentative |
| **Opportunity sampling** | Similar to convenience - use who is available at the time | Quick | Very biased |

>  **Exam Warning:** Be prepared to identify sampling methods and discuss their biases in exam questions.

---

## 1.4 Bias in Data Collection

**Sources of Bias:**

1. **Leading questions** - Wording influences response
2. **Non-response bias** - People who don't respond may have different views
3. **Sampling bias** - Sample doesn't represent population
4. **Social desirability bias** - Respondents give "acceptable" answers
5. **Interviewer bias** - Interviewer influences responses

**Minimising Bias:**
- Use random sampling where possible
- Write neutral, clear questions
- Ensure representative sample
- Guarantee anonymity for sensitive topics
- Use large enough sample size

# 2. Presenting Data (Charts and Diagrams)

## 2.1 Frequency Tables

### Discrete Data Tables

**Example:** Number of pets owned by 30 students

| Number of pets | Frequency | Frequency × value |
|----------------|-----------|-------------------|
| 0 | 8 | 0 × 8 = 0 |
| 1 | 12 | 1 × 12 = 12 |
| 2 | 7 | 2 × 7 = 14 |
| 3 | 2 | 3 × 2 = 6 |
| 4 | 1 | 4 × 1 = 4 |
| **Total** | **Σf = 30** | **Σfx = 36** |

> **Note:** This table format is essential for calculating the mean from grouped discrete data.

### Grouped Frequency Tables

Used for:
- Large range of values
- Continuous data
- Making data more manageable

**Example:** Heights of 40 students

| Height (cm) | Frequency | Midpoint (x) |
|-------------|-----------|--------------|
| 140 ≤ h < 150 | 5 | 145 |
| 150 ≤ h < 160 | 12 | 155 |
| 160 ≤ h < 170 | 15 | 165 |
| 170 ≤ h < 180 | 8 | 175 |

>  **Key Point:** When finding the mean from grouped data, use the **midpoint** of each class interval as an estimate for each value in that group.

---

## 2.2 Charts and Diagrams

### Bar Charts

**Features:**
- Used for **qualitative** or **discrete** data
- Equal width bars with **gaps** between them
- Height/frequency on vertical axis
- Categories on horizontal axis

**Construction Rules:**
1. Equal width bars
2. Gaps between bars (for discrete/qualitative data)
3. Clear axis labels
4. Title

**Dual/Composite Bar Charts:**
- Compare multiple categories side by side
- Useful for comparing groups over time

### Histograms

**Features:**
- Used for **continuous** grouped data
- **No gaps** between bars
- **Area** of bar proportional to frequency

**When class widths are equal:**
- Height directly represents frequency

**When class widths are unequal:**
- Use **frequency density**

$$\text{Frequency Density} = \frac{\text{Frequency}}{\text{Class Width}}$$

**Worked Example:**

| Time (minutes) | Frequency | Class Width | Frequency Density |
|----------------|-----------|-------------|-------------------|
| 0-10 | 15 | 10 | 15 ÷ 10 = 1.5 |
| 10-20 | 25 | 10 | 25 ÷ 10 = 2.5 |
| 20-40 | 30 | 20 | 30 ÷ 20 = 1.5 |
| 40-60 | 10 | 20 | 10 ÷ 20 = 0.5 |

**Histogram construction:**
- Plot frequency density on y-axis
- Each bar has width = class width
- Area of each bar = frequency

> 📌 **Exam Insight:** You may be asked to:
> - Complete a histogram
> - Calculate frequencies from a histogram
> - Find missing frequencies given the histogram shape

### Pie Charts

**Purpose:** Show proportions of a whole (percentages/fractions)

**Construction:**
1. Calculate total frequency
2. Find angle for each category: $\text{Angle} = \frac{\text{Frequency}}{\text{Total}} \times 360°$

**Worked Example:**

Transport to school for 72 students:
- Walk: 18 students
- Bus: 36 students
- Car: 12 students
- Cycle: 6 students

| Transport | Frequency | Calculation | Angle |
|-----------|-----------|-------------|-------|
| Walk | 18 | $\frac{18}{72} \times 360°$ | 90° |
| Bus | 36 | $\frac{36}{72} \times 360°$ | 180° |
| Car | 12 | $\frac{12}{72} \times 360°$ | 60° |
| Cycle | 6 | $\frac{6}{72} \times 360°$ | 30° |
| **Total** | **72** | | **360°** |

>  **Check:** Always verify angles sum to 360°

### Frequency Polygons

**Purpose:** Show shape of distribution for grouped continuous data

**Construction:**
1. Plot midpoint of each class interval
2. Plot frequency on y-axis
3. Join points with straight lines
4. Start and end on x-axis (if appropriate)

**Example Points:** (145, 5), (155, 12), (165, 15), (175, 8) for the height data above.

### Stem and Leaf Diagrams

**Purpose:** Display numerical data while retaining individual values

**Construction:**
1. Identify stem (leading digit(s))
2. List leaves (final digit) in order
3. Include a key

**Worked Example:**

Test scores: 34, 52, 47, 61, 55, 48, 72, 58, 41, 63, 67, 53, 59, 44

| Stem | Leaf |
|------|------|
| 3 | 4 |
| 4 | 1 4 7 8 |
| 5 | 2 3 5 8 9 |
| 6 | 1 3 7 |
| 7 | 2 |

**Key:** 3 | 4 represents 34

**Advantages:**
- Retains original data values
- Shows distribution shape
- Easy to find median (middle value)
- Easy to find mode (most common leaf)

**Back-to-back Stem and Leaf:**
- Compare two datasets
- Common stem, leaves on either side

### Cumulative Frequency Diagrams

**Purpose:** Show running total of frequencies; useful for finding medians and quartiles

**Construction:**
1. Create cumulative frequency table
2. Plot upper class boundary against cumulative frequency
3. Join points with smooth curve

**Worked Example:**

| Height (cm) | Frequency | Upper Boundary | Cumulative Frequency |
|-------------|-----------|----------------|----------------------|
| 140 ≤ h < 150 | 5 | 150 | 5 |
| 150 ≤ h < 160 | 12 | 160 | 17 |
| 160 ≤ h < 170 | 15 | 170 | 32 |
| 170 ≤ h < 180 | 8 | 180 | 40 |

**From cumulative frequency curve, find:**

| Statistic | Position | How to find |
|-----------|----------|-------------|
| Median (Q2) | $\frac{n}{2}$ | Read across from $y = 20$ |
| Lower Quartile (Q1) | $\frac{n}{4}$ | Read across from $y = 10$ |
| Upper Quartile (Q3) | $\frac{3n}{4}$ | Read across from $y = 30$ |
| Interquartile Range (IQR) | Q3 − Q1 | Difference between quartiles |

---

## 2.3 Choosing the Right Diagram

| **Data Type** | **Suitable Diagrams** |
|---------------|----------------------|
| Qualitative | Bar chart, pie chart |
| Discrete quantitative | Bar chart, stem and leaf |
| Continuous grouped | Histogram, frequency polygon, cumulative frequency |
| Comparing two datasets | Composite bar chart, back-to-back stem and leaf, dual bar chart |
| Showing trends over time | Line graph, bar chart |
| Showing proportions | Pie chart |

# 3. Misleading Data

## 3.1 How Statistics Can Mislead

### Misleading Graphs

**1. Truncated (Broken) Axes**

When the y-axis doesn't start at zero, differences appear exaggerated.

**Example:**
```
Sales Graph A (misleading):        Sales Graph B (accurate):
    |                                      |
100|    ▓▓                                |
    |    ▓▓                               |
 95|    ▓▓ ▓▓                        100| ▓▓▓▓▓▓▓▓
    |    ▓▓ ▓▓                           | ▓▓▓▓▓▓▓▓ ▓▓▓▓▓▓▓▓
 90|    ▓▓ ▓▓                        90| ▓▓▓▓▓▓▓▓ ▓▓▓▓▓▓▓▓
    +--------                              +--------
     2022 2023                              2022 2023
```

Graph A suggests a dramatic increase, but Graph B shows it's only about 10%.

**2. Inconsistent Scales**

- Unequal intervals on axes
- Missing scale values
- Distorted proportions

**3. 3D Effects**

- Make some segments appear larger
- Distort true proportions
- Often used in misleading pie charts

**4. Pictograms with Non-linear Scaling**

If symbols are scaled in **area** or **volume** instead of height:
- Double the value ≠ double the symbol size
- Creates visual distortion

>  **Exam Tip:** Always check if the area of a pictogram symbol is proportional to the value, not just the height.

### Misleading Averages

**Choosing the "convenient" average:**

**Example:** A company advertises "average salary: £35,000"
- Mean salary: £35,000
- Median salary: £25,000
- Modal salary: £22,000

The mean is inflated by a few very high salaries of executives. The median gives a more accurate picture of what "most" employees earn.

**When to be suspicious:**
- "Average" not specified (mean, median, or mode?)
- Small sample sizes
- Extreme outliers present

### Correlation vs Causation

> **Key Principle:** Just because two variables are correlated doesn't mean one causes the other.

**Examples:**
- Ice cream sales and drowning deaths (both increase in summer)
- Number of teachers and number of prisoners (both increase with population)
- Shoe size and reading ability (both increase with age)

**Spurious correlation:** A third variable explains the relationship.

### Misleading Percentages

**1. Percentage Points vs Percentage Change**

**Example:**
- Unemployment rises from 5% to 7%
- Percentage point increase: 2 percentage points
- Percentage increase: $\frac{7-5}{5} \times 100 = 40\%$

>  **Common Mistake:** Confusing percentage points with percentage change.

**2. Relative vs Absolute Change**

"A 100% increase in risk" sounds alarming, but:
- Risk rising from 1 in 1,000,000 to 2 in 1,000,000
- Still extremely rare!

**3. Base Effects**

"If prices fall 50% then rise 50%, you're back where you started"

❌ **False:** 
- Original price: £100
- After 50% fall: £50
- After 50% rise: £75

### Sampling Bias in Headlines

**Examples:**
- "9 out of 10 dentists recommend..."
  - Which dentists? How were they selected?
- "Users rate our product 5 stars"
  - Only satisfied customers leave reviews

---

## 3.2 Evaluating Statistical Claims

**Questions to ask:**

1. **Source:** Who collected the data? Do they have an agenda?
2. **Sample:** How large? How was it selected?
3. **Method:** How was data collected? Any bias?
4. **Presentation:** Is the graph misleading? Does the average chosen misrepresent the data?
5. **Context:** What's missing? What's being compared?

> 📌 **Exam Focus:** You may be asked to identify what is misleading about a graph or statistical claim, or to suggest improvements.

# 4. Averages (Mean, Median, Mode)

## 4.1 The Three Averages

### Mean

> **Definition:** The sum of all values divided by the number of values.

**Formula:**
$$\text{Mean} = \frac{\text{Sum of values}}{\text{Number of values}} = \frac{\Sigma x}{n}$$

**For frequency data:**
$$\text{Mean} = \frac{\Sigma fx}{\Sigma f}$$

where $f$ is frequency and $x$ is the value.

**Properties:**
- Uses **all** values
- Affected by **outliers** (extreme values)
- May not be an actual data value
- Useful for further statistical calculations

**Worked Example 1 (Raw data):**

Find the mean of: 12, 8, 15, 10, 20

$$\text{Mean} = \frac{12 + 8 + 15 + 10 + 20}{5} = \frac{65}{5} = 13$$

**Worked Example 2 (Frequency table):**

| Score (x) | Frequency (f) | fx |
|-----------|---------------|-----|
| 0 | 3 | 0 |
| 1 | 5 | 5 |
| 2 | 8 | 16 |
| 3 | 4 | 12 |
| **Total** | **Σf = 20** | **Σfx = 33** |

$$\text{Mean} = \frac{33}{20} = 1.65$$

### Median

> **Definition:** The middle value when data is arranged in order.

**Finding the position of the median:**
- For $n$ values, median is at position $\frac{n + 1}{2}$

**Worked Example:**

Find the median of: 7, 12, 3, 15, 9, 4, 11

**Step 1:** Arrange in order: 3, 4, 7, 9, 11, 12, 15

**Step 2:** Find position: $\frac{7 + 1}{2} = 4^{th}$ value

**Step 3:** Median = 9

**For even number of values:**

Find the median of: 3, 4, 7, 9, 11, 12

Position: $\frac{6 + 1}{2} = 3.5^{th}$ value

Take average of 3rd and 4th values:
$$\text{Median} = \frac{7 + 9}{2} = 8$$

**Properties:**
- Not affected by outliers
- Always a value that exists in the data (for odd n)
- Useful when extreme values would distort the mean

### Mode

> **Definition:** The most frequently occurring value.

**Worked Example:**

Find the mode of: 5, 7, 3, 7, 8, 5, 7, 2, 9

**Step 1:** Count frequencies:
- 2: appears once
- 3: appears once
- 5: appears twice
- 7: appears three times ← most frequent
- 8: appears once
- 9: appears once

**Mode = 7**

**Types of modes:**
- **Unimodal:** One mode
- **Bimodal:** Two modes
- **Multimodal:** More than two modes
- **No mode:** All values appear equally often

**Properties:**
- Can be used for qualitative data
- Not affected by outliers
- May not exist or may have multiple values

---

## 4.2 Choosing the Appropriate Average

| **Situation** | **Best Average** | **Reason** |
|---------------|------------------|------------|
| No extreme values, symmetric distribution | Mean | Represents typical value well |
| Extreme outliers present | Median | Not affected by extremes |
| Categorical/qualitative data | Mode | Only option for non-numerical data |
| Most common value is needed | Mode | Shows most typical response |
| Shoe sizes to stock | Mode | Stock what sells most |
| Income distribution | Median | High earners skew the mean |

**Worked Example - Effect of Outliers:**

Data: 25, 27, 28, 30, 32, 35, 150

- Mean: $\frac{327}{7} = 46.7$ (distorted by outlier)
- Median: 30 (not affected)
- Mode: None

The median (30) better represents the typical value.

---

## 4.3 Mean from Grouped Data

When data is grouped, we **estimate** the mean using midpoints.

**Formula:**
$$\text{Estimated Mean} = \frac{\Sigma fx}{\Sigma f}$$

where $f$ = frequency and $x$ = class midpoint

**Worked Example:**

| Mass (kg) | Frequency (f) | Midpoint (x) | fx |
|-----------|---------------|--------------|-----|
| 40 ≤ m < 50 | 6 | 45 | 270 |
| 50 ≤ m < 60 | 14 | 55 | 770 |
| 60 ≤ m < 70 | 18 | 65 | 1170 |
| 70 ≤ m < 80 | 8 | 75 | 600 |
| 80 ≤ m < 90 | 4 | 85 | 340 |
| **Total** | **Σf = 50** | | **Σfx = 3150** |

$$\text{Estimated Mean} = \frac{3150}{50} = 63 \text{ kg}$$

>  **Note:** This is an **estimate** because we assume all values in each group are at the midpoint.

---

## 4.4 Median from Grouped Data

### Method 1: From Cumulative Frequency Curve

1. Find $\frac{n}{2}$ on the cumulative frequency axis
2. Read across to the curve
3. Read down to find the median value

### Method 2: Linear Interpolation

**Formula:**
$$\text{Median} = L + \left(\frac{\frac{n}{2} - F}{f}\right) \times w$$

Where:
- $L$ = lower boundary of median class
- $n$ = total frequency
- $F$ = cumulative frequency before median class
- $f$ = frequency of median class
- $w$ = class width

**Worked Example:**

Using the mass data above (n = 50, median position = 25th):

| Mass (kg) | Frequency | Cumulative Frequency |
|-----------|-----------|----------------------|
| 40 ≤ m < 50 | 6 | 6 |
| 50 ≤ m < 60 | 14 | 20 |
| 60 ≤ m < 70 | 18 | 38 ← median class (25 is in here) |
| 70 ≤ m < 80 | 8 | 46 |
| 80 ≤ m < 90 | 4 | 50 |

- $L = 60$
- $n = 50$, so $\frac{n}{2} = 25$
- $F = 20$ (cumulative frequency before median class)
- $f = 18$ (frequency of median class)
- $w = 10$

$$\text{Median} = 60 + \left(\frac{25 - 20}{18}\right) \times 10$$

$$= 60 + \frac{5}{18} \times 10$$

$$= 60 + 2.78 = 62.78 \text{ kg (to 2 d.p.)}$$

---

## 4.5 Modal Class

For grouped data, we find the **modal class** (the class with the highest frequency) rather than a single mode.

**From the mass data above:**

The highest frequency is 18, corresponding to the class **60 ≤ m < 70**.

Modal class: **60 ≤ m < 70 kg**

> 📌 **Exam Note:** When asked for the mode from grouped data, always state the **modal class**, not a single value.

---

## 4.6 Range and Measures of Spread

### Range

$$\text{Range} = \text{Maximum value} - \text{Minimum value}$$

**Limitation:** Affected by outliers

### Interquartile Range (IQR)

$$\text{IQR} = Q_3 - Q_1$$

where:
- $Q_1$ = lower quartile (25th percentile)
- $Q_3$ = upper quartile (75th percentile)

**Finding quartiles from cumulative frequency:**
- $Q_1$: Position $\frac{n}{4}$
- $Q_3$: Position $\frac{3n}{4}$

**Advantages:** Not affected by outliers, gives spread of middle 50% of data

**Worked Example:**

From the cumulative frequency curve (n = 50):
- $Q_1$ at position $\frac{50}{4} = 12.5$: approximately 54 kg
- $Q_3$ at position $\frac{3 \times 50}{4} = 37.5$: approximately 71 kg

$$\text{IQR} = 71 - 54 = 17 \text{ kg}$$

---

## 4.7 Comparing Distributions

When comparing two datasets, comment on:

1. **A measure of location** (mean or median)
2. **A measure of spread** (range or IQR)
3. **Shape** (if visible from diagram)

**Example Response:**

"The median height of girls (165 cm) is higher than the median height of boys (160 cm), suggesting girls are generally taller. However, the IQR for boys (20 cm) is greater than for girls (12 cm), showing greater variability in boys' heights."

>  **Exam Tip:** Always compare using context - don't just state numbers without interpretation.

## 5.3 Gaining Full Marks

### For Calculation Questions:

✓ Show all working clearly
✓ Use correct formulas
✓ Check your answer is reasonable
✓ Give units where appropriate
✓ Round to appropriate accuracy (usually same as data or 3 significant figures)

### For Graph Questions:

✓ Use proper scales with equal intervals
✓ Label axes clearly
✓ Include title
✓ For histograms: remember frequency density for unequal class widths
✓ For pie charts: check angles sum to 360°

### For Interpretation Questions:

✓ Use data values in your answer
✓ Refer to context (not just "the data shows...")
✓ Use comparative language (higher, lower, greater, less)
✓ Be specific (quote values)

### Common Mark Scheme Patterns:

| **Marks** | **What's Expected** |
|-----------|---------------------|
| 1 mark | Single value or simple statement |
| 2 marks | Method and answer, or two linked points |
| 3 marks | Full method, answer, and interpretation |
| 4+ marks | Complete analysis with multiple steps and interpretation |

---

## 5.4 Common Pitfalls to Avoid

1. ❌ **Confusing mean, median, and mode** - Know when to use each
2. ❌ **Forgetting frequency in grouped mean** - Use $\Sigma fx$, not just $\Sigma x$
3. ❌ **Using class boundaries instead of midpoints** for grouped mean
4. ❌ **Drawing bars with gaps in histograms** - Continuous data has no gaps
5. ❌ **Not using frequency density** for histograms with unequal class widths
6. ❌ **Confusing percentage change with percentage points**
7. ❌ **Not checking pie chart angles sum to 360°**
8. ❌ **Giving a single mode for grouped data** - State modal class instead
9. ❌ **Not interpreting in context** - Always relate to the situation
10. ❌ **Using mean with extreme outliers** - Consider median instead
