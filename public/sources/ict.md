# Chapter 1: Digital Devices

Digital devices are pieces of hardware that use computers or microcontrollers to create, store, and process data. They support how we live our lives every day by connecting and working together to provide the data we need.

## Types of Digital Devices
- **Mainframe Computers**
    - Large, powerful computers used by massive organisations.
    - Designed for complex processing tasks like statistical analysis.
    - Used for bulk data processing.
    - Can be used by a lot of people at the same time.
    - Highly expensive to purchase and maintain.
- **Server**
    - High-capacity computer providing services to other computers.
    - Handles tasks like file storage, hosting websites, and managing network traffic.
    - Usually runs 24/7 without being shut down.
- **Personal Computers (PCs)**
    - **Desktops**:
        - Designed to be used at a single location (desk).
        - Usually have more space for components than laptops.
        - Easier for users to upgrade or add additional components.
        - Require peripheral devices like a monitor, keyboard, and mouse to be connected.
        - **All-in-One**: Combines the monitor with the PC hardware into a single unit.
    - **Laptops**:
        - Include a keyboard, screen, trackpad, and rechargeable battery.
        - Highly portable devices that can be moved away from a desk.
        - **Desktop Replacements**: Larger laptops with high performance and bigger screens.
- **Microprocessors**
    - Small devices that control what a computer does.
    - Found in household appliances like washing machines.
    - Embedded into televisions and microwave ovens.
    - Take data as input, process it, and provide an output.
- **Single-Board Computers (SBCs)**
    - Affordable computers used in education and physical computing projects.
    - Example: **Raspberry Pi Zero**.
    - Use cheap microSD cards for storage.
    - Very small and consume very little power.
- **Mobile Phones**
    - **Smartphones**: Multifunctional devices with internet access and high portability.
    - **Specialist Phones**:
        - Features like emergency buttons for the elderly.
        - Accessibility features for those with disabilities (e.g., Alto 2 'talking phone' for the blind).
        - Connect to networks via **SIM (Subscriber Identity Module)** cards.
- **Tablet Devices**
    - Touchscreen-based portable devices.
    - Larger than smartphones but smaller than laptops.
    - Used heavily for media consumption and light productivity.
- **Digital Cameras and Camcorders**
    - Capture images and video as digital data.
    - Use high-quality sensors to convert light into pixels.
    - Store data on flash memory cards (e.g., SD cards).
- **Games Consoles**
    - Specialized computers designed for high-performance video gaming.
    - Feature powerful graphics processors (GPUs).
    - Support peripherals like controllers and headsets.
- **Home Entertainment Systems**
    - Include Smart TVs, Blu-ray players, and Soundbars.
    - Provide high-definition audio and visual experiences.
- **Media Players**
    - Portable devices for playing audio and video (e.g., MP3 players).
    - Can store thousands of files on internal flash storage.
- **Navigation Aids (GPS)**
    - Used to calculate position using satellite signals.
    - Crucial for maps, tracking, and logistics.

## Input Devices
Input devices carry data from the outside world into the computer system.

| Device | Description | Primary Use Case |
| :--- | :--- | :--- |
| **Keyboard** | Set of keys for entering text. | Typing reports and emails. |
| **Mouse** | Point-and-click pointing device. | Navigating Graphical User Interfaces. |

| **Joystick** | Lever that can move in all directions. | Controlling planes in flight simulators. |
| **Graphics Tablet** | Flat pad used with a stylus (pen). | Digital illustration and graphic design. |
| **Scanner** | Converts physical documents to digital. | Archiving old photos or paper records. |
| **Digital Camera** | Captures photos digitally. | Professional photography and news. |
| **Webcam** | Small camera for video calls. | Virtual meetings and streaming. |
| **Microphone** | Converts sound into digital signals. | Voice commands and recording audio. |
| **Touch Screen** | Display that senses touch. | Smartphones and public kiosks. |
| **OMR Reader** | Optical Mark Recognition. | Grading multiple-choice exam papers. |
| **OCR Reader** | Optical Character Recognition. | Converting scanned text into editable data. |
| **Bar Code Scanner** | Reads printed blocks of lines. | Supermarket checkouts and stock control. |

    - **Temperature**: Used in greenhouses to open windows.
    - **Moisture**: Used in farming to trigger irrigation systems.
    - **Light**: Turns on street lamps at dusk.
    - **Pressure**: Triggers burglar alarms when someone stands on a mat.
    - **Proximity**: Helps cars park without hitting objects.

## Output Devices
Output devices present processed data to the user.

- **Monitors**
    - **Screen Size**: Measured diagonally in inches.
    - **Resolution**: Number of pixels on the screen (Width x Height). Higher = Sharper.
- **Data Projector**
    - Large-scale output for presentations in halls or classrooms.
- **Speakers**
    - Output for audio signals, music, and voice alerts.
- **Control Devices (Actuators)**
    - Physical outputs like motors, pumps, and heaters controlled by the computer.
- **Printers (The 4 Main Types)**
    - **Dot Matrix Printer**:
        - Uses a series of pins to create dots on paper.
        - Simple and reliable, but slow and noisy.
        - Good for receipts and labels.
    - **Laser Printer**:
        - Uses a laser beam to create static electricity on a drum.
        - Toner (powder) is attracted to the drum and transferred to paper.
        - Very fast and produces high-quality text.
        - High initial cost but low cost per page.
    - **Inkjet Printer**:
        - Sprays tiny droplets of liquid ink onto the paper.
        - Excellent for high-quality photographs.
        - Cheaper to buy initially but expensive ink cartridges.
        - Slower than laser printers.
    - **3D Printer**:
        - Builds physical objects layer by layer using materials like plastic (PLA/ABS).
        - Used for creating prototypes, medical implants, and jewelry.
        - Requires specialized software and 3D models.
    - **Plotter**:
        - Uses a pen to draw continuous lines on very large sheets of paper.
        - Essential for blueprints, architectural plans, and engineering drawings.
        - Much slower than other printers but highly accurate for large scales.

## Storage Media and Devices
Storage devices are the hardware that reads/writes data. Storage media is the physical material.

- **Storage Capacity Hierarchy**:
    - **Bit**: 1 or 0 (Binary digit).
    - **Byte**: 8 bits.
    - **Kilobyte (KB)**: 1000 bytes.
    - **Megabyte (MB)**: 1000 KB.
    - **Gigabyte (GB)**: 1000 MB.
    - **Terabyte (TB)**: 1000 GB.

- **Magnetic Storage**
    - **Hard Disk Drive (HDD)**:
        - Uses spinning magnetic platters and a read/write head.
        - High capacity (up to 20TB+).
        - Cheap per GB.
        - *Weakness*: Moving parts make it fragile and slower than SSDs.
    - **Magnetic Tape**:
        - Sequential access (must rewind to find data).
        - Extremely high capacity and very low cost.
        - Used by companies for long-term backups (archiving).

- **Solid State Storage**
    - **Solid State Drive (SSD)**:
        - No moving parts, using flash memory chips.
        - Extremely fast read/write speeds.
        - Silent, cool, and highly durable.
        - *Weakness*: More expensive than HDDs per GB.
    - **Flash Memory (USB Sticks / SD Cards)**:
        - Portable and small.
        - Used in cameras, phones, and for transferring files.

- **Optical Storage**
    - Uses lasers to read/write pits and lands on the disk surface.
    - **CD (Compact Disc)**: 700MB capacity.
    - **DVD (Digital Versatile Disc)**: 4.7GB capacity.
    - **Blu-ray**: 25GB to 50GB+ capacity, used for high-def movies and games.
    - **Types**:
        - **-R (Recordable)**: Write once, read many times.
        - **-RW (Re-writable)**: Can be erased and written to many times.

## Accessibility in Digital Devices
- ICT provides critical support for people with varying needs.
- **Visually Impaired**:
    - Screen Magnifiers (Software).
    - Braille Keyboards (Hardware).
    - Screen Readers (Software - text to speech).
- **Hearing Impaired**:
    - Subtitles and closed captions.
    - Visual alerts instead of sound beeps.
- **Physical Disabilities**:
    - Eye-tracking systems.
    - Mouth sticks for typing.
    - Foot-controlled mice.

## Past Paper Questions

**Q1. Evaluate the impact of using a tablet computer instead of a laptop for a student. (8 marks)**
- **1. Weight (+1)**: Tablets are significantly lighter, reducing the physical strain of carrying a device between classes.
- **2. Battery Life (+1)**: Tablets often have longer battery life than high-performance laptops, lasting a full school day on one charge.
- **3. Interactive Learning (+1)**: The touchscreen and stylus support allow for direct annotation of diagrams and hand-drawn notes.
- **4. Camera Use (+1)**: Tablets are easier to use for taking photos of experiments or whiteboards due to their portability.
- **5. Cost (+1)**: Entry-level tablets are often cheaper than full-featured laptops, making them more accessible to students.
- **6. Typing Speed (-1)**: On-screen keyboards are slower and more prone to errors than physical laptop keyboards for long essays.
- **7. Software Limitations (-1)**: Tablet operating systems (iPadOS/Android) might not support full professional versions of complex software.
- **8. Connectivity (-1)**: Tablets have fewer physical ports (USB/HDMI) compared to laptops for connecting printers or monitors.
- **9. Fragility (-1)**: Large thin screens on tablets are easily cracked if dropped without a heavy-duty case.
- **10. Multitasking (-1)**: It is harder to view multiple windows or complex side-by-side documents on a tablet interface.

**Q2. Discuss the advantages and disadvantages of a company transitioning from HDD to SSD for all desktop computers. (8 marks)**
- **1. Speed (+1)**: SSDs allow the operating system and applications to boot almost instantly, increasing employee productivity.
- **2. Durability (+1)**: Since there are no moving parts, the risk of data loss from physical shocks or vibrations is greatly reduced.
- **3. Efficiency (+1)**: SSDs consume less power and generate less heat, potentially lowering the company's energy bills and cooling costs.
- **4. Noise (+1)**: SSDs are silent, creating a quieter and more pleasant office environment for concentrated work.
- **5. Reliability (+1)**: SSDs have a lower failure rate than HDDs, meaning fewer hardware replacements and less downtime for IT staff.
- **6. Initial Cost (-1)**: Buying hundreds of SSDs is much more expensive upfront than purchasing the same capacity in HDDs.
- **7. Capacity (-1)**: High-capacity SSDs (4TB+) are extremely expensive, so the company might have to limit local storage space.
- **8. Write Cycles (-1)**: SSDs have a finite number of times data can be written to them before the flash memory wears out.
- **9. Data Recovery (-1)**: It is often much harder and more expensive to recover lost data from a failed SSD than from a magnetic HDD.
- **10. Legacy Support (-1)**: Older office hardware might require adapters or not fully support the high speeds of modern SATA/NVMe SSDs.

**Q3. Evaluate the use of 3D printers in the healthcare industry. (8 marks)**
- **1. Customization (+1)**: Prosthetic limbs can be printed to the exact millimetre of a patient's body, providing a perfect fit.
- **2. Prototyping (+1)**: Dentists can quickly print models of teeth to test braces or crowns before final manufacturing.
- **3. Education (+1)**: Surgeons can print 3D models of a patient's specific tumour based on MRI scans to practice a surgery.
- **4. Cost reduction (+1)**: Local printing of medical tools reduces the need for expensive shipping and international logistics.
- **5. Organ Printing (+1)**: Emerging "Bioprinting" technology could eventually print replacement tissues using the patient's own cells.
- **6. Material Strength (-1)**: Printed materials may not yet match the long-term durability of traditional surgical steel or titanium.
- **7. Certification (-1)**: It is difficult for regulators to ensure every printed medical device meets strict safety standards.
- **8. Production Speed (-1)**: 3D printing is a slow process; it cannot yet mass-produce items as fast as injection moulding.
- **9. Skill Gap (-1)**: Medical staff require specialized training to use CAD software and manage complex 3D printers correctly.
- **10. Biocompatibility (-1)**: Some resins or plastics used in printing can cause allergic reactions or be toxic if left inside the body.

## Where you can lose marks
- **Magnetic vs Optical**: Don't confuse them! Magnetic uses **magnets and platters**; Optical uses **lasers and pits**.
- **SSD Speed**: Never say "it makes the internet faster"; it only makes **local file access** and **boot times** faster.
- **Resolution**: Don't just say 'quality'; use the term **'Total number of pixels on the screen'**.
- **Mainframe vs Server**: A mainframe is one **giant, powerful** unit; a server is often part of a network and can be a standard high-spec PC.
- **ROM vs RAM**: RAM is **Volatile** (lost at power off); ROM is **Non-Volatile** (kept at power off). Never say "temporary" vs "permanent" - use the technical terms for full marks.
- **GPS**: A device needs **4 satellites** for an accurate 3D position (including altitude), though 3 can give a 2D position. Stick to 4 for the exam.

# Chapter 2: Software

Software consists of the programs and instructions that tell the hardware how to function. It is the "brain" behind the physical components.

## Categories of Software
Software is broadly divided into two main categories:

### 1. System Software
Software that manages the hardware and provides a platform for application software to run.
- **Operating System (OS)**:
    - Manages RAM allocation and processor time.
    - Handles input and output devices.
    - Provides the **User Interface**.
    - Manages file storage and security permissions.
- **Utility Software**:
    - **Anti-virus**: Scans for and removes malicious code.
    - **Backup Software**: Creates copies of data to prevent loss.
    - **Compression**: Reduces file size by removing redundant data (Lossy/Lossless).
    - **Defragmenter**: Reorganizes files on a hard drive for faster access.
- **Device Drivers**:
    - Small programs that allow the OS to communicate with specific hardware (e.g., a printer).

### 2. Application Software
Software that allows the user to perform specific tasks.
- **Word Processors**: Creating typed documents (e.g., MS Word).
- **Spreadsheets**: Calculations and data modeling (e.g., Excel).
- **Databases**: Storing and searching large records (e.g., Access).
- **Web Browsers**: Accessing and viewing websites (e.g., Chrome).
- **Graphics Editors**: Creating and editing images (e.g., Photoshop).

## User Interfaces
| Type | Description | Best For... |
| :--- | :--- | :--- |
| **GUI (Graphical User Interface)** | Uses icons, menus, and windows. Controlled via mouse/touch. | Everyday users, children, and office work. |
| **CLI (Command Line Interface)** | User types text commands into a prompt. | System administrators, experts, and running scripts. |
| **Menu-Driven Interface** | User selects options from a list (e.g., ATM). | Public kiosks and simple devices. |
| **Voice Interface** | Recognizes spoken commands (e.g., Siri). | Hands-free use and accessibility for the blind. |

## Open Source vs Proprietary Software
| Feature | Open Source | Proprietary |
| :--- | :--- | :--- |
| **Source Code** | Publicly available for anyone to edit. | Secret and owned by a company. |
| **Cost** | Usually free. | Usually requires a paid license/subscription. |
| **Support** | Community-driven forums and wikis. | Official professional helpdesk and patches. |
| **Examples** | Linux, Android, LibreOffice. | Windows, MacOS, MS Office. |

## Software Licensing
- **Individual License**: One user/device only.
- **Multi-user License**: A specific number of users allowed.
- **Site License**: Everyone at a specific physical location (e.g., a school).
- **EULA (End-User License Agreement)**: The legal contract you accept when installing software.

# Chapter 3: Memory and Processors

The Central Processing Unit (CPU) and Memory work together to execute instructions and process data in a cycle.

## The Central Processing Unit (CPU)
The CPU is the hardware that executes programs and manages the rest of the computer's hardware.

- **Control Unit (CU)**:
    - Coordinates the activities of the CPU.
    - Manages the Fetch-Decode-Execute cycle.
    - Sends signals to other components.
- **Arithmetic Logic Unit (ALU)**:
    - Performs mathematical calculations (+, -, *, /).
    - Performs logical comparisons (AND, OR, NOT, <, >).
- **Registers**:
    - Tiny, extremely fast storage locations inside the CPU.
    - Used to hold data currently being processed.

## Factors Affecting CPU Performance
| Factor | Description | Impact |
| :--- | :--- | :--- |
| **Clock Speed** | The number of cycles per second (Hertz). | Higher speed = More instructions processed per second. |
| **Number of Cores** | Multiple independent processors in one chip. | More cores = Better multitasking (parallel processing). |
| **Cache Size** | Fast memory located very close to the CPU. | Larger cache = Less time wasted fetching from slow RAM. |
| **Bus Width** | The number of wires used to transfer data. | Wider bus = More data transferred in a single clock tick. |

## Primary Memory
Memory that the CPU can access directly and quickly.

- **RAM (Random Access Memory)**:
    - **Volatile**: All data is lost when power is turned off.
    - Used to store the OS, running apps, and current data.
    - Higher RAM allows for more programs to run simultaneously without slowdown.
- **ROM (Read Only Memory)**:
    - **Non-Volatile**: Keeps data when the power is off.
    - Contains the **BIOS / Bootloader** (instructions to start the computer).
    - Cannot be written to by normal users.
- **Cache Memory**:
    - Faster than RAM but smaller in capacity.
    - Stores frequently used data to speed up the CPU cycle.

## Secondary Storage vs Primary Memory
- **Primary**: Fast, low capacity, expensive per byte, directly accessible by CPU.
- **Secondary**: Slower, high capacity, cheap per byte, used for long-term storage of files.

## Specialized Processors
- **GPU (Graphics Processing Unit)**:
    - Designed specifically to handle visual data and complex math for 3D rendering.
    - Frees up the CPU for other tasks.
    - Essential for gaming, video editing, and AI training.

## Past Paper Questions

**Q1. Discuss how the combination of Clock Speed, Cores, and Cache determines a computer's overall speed. (8 marks)**
- **1. Basic Processing (+1)**: High clock speed (e.g., 4GHz) ensures the Fetch-Execute cycle runs as fast as possible for a single task.
- **2. Parallelism (+1)**: Having multiple cores (e.g., Quad-core) allows the computer to process different parts of a program at the same time.
- **3. Multitasking (+1)**: Multiple cores enable several heavy apps (e.g., a game and a recorder) to run smoothly without competing for one cycle.
- **4. Data Access (+1)**: Cache acts as a buffer; if the data is in the cache, the CPU doesn't have to wait for the much slower RAM.
- **5. Efficiency (+1)**: A large cache reduces the "Idle" time of a fast processor, making the clock speed more effective.
- **6. Heat Issues (-1)**: Increasing clock speed (Overclocking) generates massive heat, which can damage the chip or cause "thermal throttling."
- **7. Software Limits (-1)**: Many programs are not designed for "Multi-threading," so extra cores might sit idle while one core does all the work.
- **8. Cost (-1)**: High-end CPUs with massive cache and many cores are significantly more expensive, yielding diminishing returns for basic users.
- **9. Bottlenecks (-1)**: Even with a fast CPU, a slow HDD or low RAM will "bottleneck" the system, making the CPU wait for data.
- **10. Optimization (+1)**: A balanced system with moderate cores and high cache is often better than an unbalanced system with one ultra-fast spec.

**Q2. Evaluate the importance of RAM versus Secondary Storage for a person editing 4K video. (8 marks)**
- **1. Real-time Preview (+1)**: High RAM allows the editing software to hold the video data in memory for smooth, lag-free playback while editing.
- **2. Workload Management (+1)**: Large files in RAM mean the system doesn't have to constantly "swap" data to the slower disk (Virtual Memory).
- **3. Application Stability (+1)**: Professional video apps are memory-intensive; low RAM can lead to frequent crashes during complex renders.
- **4. Archiving Files (+1)**: High-capacity secondary storage (HDDs) is essential for storing the massive multi-terabyte raw video footage.
- **5. Export Speed (+1)**: Reading data from an SSD (Secondary Storage) during export is much faster than from a magnetic HDD.
- **6. File Permanence (+1)**: Secondary storage is non-volatile, meaning the project is safe when the computer is turned off.
- **7. Volatility Limit (-1)**: Data in RAM is risky; if the power fails before saving to secondary storage, hours of work are lost.
- **8. Cost Balance (-1)**: RAM is much more expensive per GB, so editors must balance high RAM for editing with high disk space for storage.
- **9. Transfer Latency (-1)**: Even with 128GB of RAM, if the secondary storage is slow, it takes a long time to initially "load" the project into RAM.
- **10. Portable Storage (+1)**: Secondary storage like external SSDs allows the editor to move large projects between different workstations easily.

---

## Where you can lose marks
- **Volatile**: Use this exact word for RAM. It means **lost when power is disconnected**.
- **Fetch-Execute Cycle**: Mention the stages: 1. Fetch from RAM, 2. Decode in CU, 3. Execute in ALU.
- **Cores**: Don't say "it makes the computer twice as fast"; say "it allows for better **simultaneous processing** of multiple tasks."
- **Overclocking**: Explain it as **increasing the clock speed above its base frequency**.
- **GPU**: It isn't just for gaming; it is for **intensive mathematical calculations** and **visual manipulation**.

# Chapter 4: Digital Communication 

Digital communication is the method by which data is transferred between devices across networks, ranging from short-range personal connections to global satellite links.

## Bandwidth and Latency
The performance of any communication link is determined by two primary metrics:
- **Bandwidth**:
    - The measure of how much data can move across a connection in one second.
    - Measured in **Bits per second (bps)**, Kbps, Mbps, or Gbps.
    - Think of it as a pipe; the wider the pipe, the more data "water" flows through.
    - High bandwidth is required for:
        - Streaming 4K video.
        - Large file downloads.
        - High-definition video conferencing.
- **Latency**:
    - The time delay (lag) between a request for data and the data being received.
    - Measured in milliseconds (ms).
    - Even with high bandwidth, high latency can make a connection feel slow.
    - Critical for:
        - Real-time online gaming.
        - VoIP (Voice over Internet Protocol) calls.
        - Stock market trading.

## Factors Affecting Transfer Speed
1.  **Transfer Method**: Fibre optic is much faster than copper or wireless radio waves.
2.  **Infrastructure**: Old, degraded cables or low-quality routers reduce effective bandwidth.
3.  **Network Congestion**: Many users sharing the same connection (e.g., peak evening hours).
4.  **Signal Interference**: Electronic devices or power lines disrupting wireless or unshielded wired signals.
5.  **Distance**: Signal strength drops (attenuates) over longer distances.
6.  **Physical Blockages**: Walls, trees, and buildings stop or weaken Wi-Fi and satellite signals.

## Wired Connectivity
| Type | Description | Pro | Con |
| :--- | :--- | :--- | :--- |
| **Ethernet (Copper/UTP)** | Signal sent as electrical pulses over copper wires. | Cheap and compatible with almost everything. | Limited distance (100m) and prone to interference. |
| **Fibre Optic** | Signal sent as light pulses through glass fibres. | Extremely high speeds over long distances. | Very expensive to install and fragile. |
| **USB/Thunderbolt** | High-speed direct connection between devices. | Power delivery and data in one cable. | Very short range (usually 1-2 metres). |
| **HDMI** | Sends high-def video and audio signals. | Single cable for both media types. | Mainly for output, not general networking. |

## Wireless Connectivity
| Type | Description | Range | Use Case |
| :--- | :--- | :--- | :--- |
| **Wi-Fi** | Uses radio waves to connect devices to a router. | Up to 100m. | Home and office internet browsing. |
| **Bluetooth** | Short-range radio connection for small data. | 10m. | Wireless headphones, keyboards, mice. |
| **4G / 5G (Cellular)** | High-speed mobile data from masts. | Many kilometres. | Using internet while on the move. |
| **Satellite** | Signal sent from Earth to orbital satellites. | Global. | Internet in rural areas or ships at sea. |
| **NFC (Near Field)** | Extremely close proximity communication. | 4cm. | Contactless payments (Apple/Google Pay). |

## Network Types
- **PAN (Personal Area Network)**:
    - Connecting devices around an individual (e.g., phone to smartwatch via Bluetooth).
- **LAN (Local Area Network)**:
    - Devices connected in a small area like a house, school, or office.
- **WAN (Wide Area Network)**:
    - Connects LANs over large distances (countries/world). The Internet is the largest WAN.
- **VPN (Virtual Private Network)**:
    - A secure, encrypted tunnel through a public network (like the Internet).
    - Allows remote workers to access their office LAN securely.
- **Tethering (Hotspot)**:
    - Using a smartphone's 4G/5G connection to provide Wi-Fi to other devices.

## Specialized Communication
- **GPS (Global Positioning System)**:
    - A network of satellites that broadcast timed signals.
    - Receivers (phones, cars) calculate their exact location on Earth.
- **Broadcasting**:
    - **DVB (Digital Video Broadcasting)**: Modern TV standard.
    - **DAB (Digital Audio Broadcasting)**: High-quality radio with text info.
- **Streaming**:
    - Playing media while it is being downloaded (buffer).
    - Requires consistent bandwidth to avoid stutters.

# Chapter 6: Risks to Data and Personal Information

As we become more dependent on digital devices, the risks to our data and personal safety increase. Security is a combination of technical tools, physical barriers, and human awareness.

## Access Control
The first line of defense is ensuring only authorized people can use a system.
- **Identification**: The process of a user claiming an identity (e.g., entering a username).
- **Authentication**: The process of proving that identity (e.g., entering a password).

### Multi-Factor Authentication (MFA)
MFA requires two or more pieces of evidence from different categories:
1.  **Something You Know**: (e.g., Password, PIN, Security question).
2.  **Something You Have**: (e.g., A security token, a smartphone for an SMS code, a smart card).
3.  **Something You Are**: (e.g., Biometrics like fingerprints, iris scans, or facial recognition).

## Malware (Malicious Software)
Malware is any software designed to disrupt, damage, or gain unauthorized access to a computer system.

| Type | How it Works | Unique Characteristic |
| :--- | :--- | :--- |
| **Virus** | Attaches itself to a legitimate file or program. | **Needs a host** and human action (opening the file) to spread. |
| **Worm** | Exploits system vulnerabilities to copy itself across a network. | **Self-spreading** and does not need a host file. |
| **Trojan Horse** | Disguises itself as useful or legitimate software. | Does not replicate; relies on tricking the user. |
| **Ransomware** | Encrypts all user files and demands payment. | Extorts the victim using cryptography. |
| **Spyware** | Secretly monitors user activity (e.g., keyloggers). | Sends captured data to a hacker without detection. |
| **Adware** | Automatically displays unwanted advertisements. | Primarily designed to generate revenue through pop-ups. |
| **Rootkit** | Gains high-level (admin) access while hiding from the OS. | Can bypass security software by hiding deep in the system. |
| **Botnet** | A network of infected "zombie" computers. | Controlled by a central "bot herder" for DDoS attacks. |
| **Bug** | A mistake or flaw in a program's code. | Often creates a "loophole" or "vulnerability" that hackers exploit. |

## Social Engineering
Social engineering exploits human psychology rather than technical flaws.
- **Phishing**: Mass emails pretending to be a bank or service to steal credentials.
- **Pretexting**: Creating a fake scenario (e.g., pretending to be an IT worker needing a password).
- **Quid Pro Quo**: Offering a service (e.g., "I'll fix your laptop") in exchange for login info.
- **Shoulder Surfing**: Physically watching someone enter a PIN or password.
- **Baiting**: Leaving an infected USB stick in a public place, hoping someone plugs it in.
- **Tailgating / Piggybacking**: Following an authorized person through a secure door.
- **Pharmng**: Redirecting a user to a fake website even if they type the correct URL (via DNS poisoning).

## Technical Security Methods
- **Firewalls**:
    - Hardware or software that filters network traffic.
    - Blocks unauthorized incoming data based on a set of rules.
- **Encryption**:
    - Scrambling data using an algorithm so it is unreadable without a key.
    - **Symmetric**: Same key for encrypting and decrypting.
    - **Asymmetric**: Different keys (Public/Private) used together.
- **Anti-Malware Software**:
    - Regularly scans files for "definitions" (signatures) of known malware.
    - Quarantines or deletes suspicious files.
- **Access Rights (Permissions)**:
    - Setting rules on who can **Read**, **Write**, or **Execute** specific files.

## Physical Security
- **Biometric Locks**: Fingerprint or retina scans to enter server rooms.
- **CCTV**: Deterrent and evidence-gathering tool for physical break-ins.
- **Alarms**: Detect sound, movement, or heat in restricted areas.
- **Locked Cabinets**: Securing server racks and backup media.
- **Privacy Screens**: Filters on monitors that prevent viewing from an angle (stops shoulder surfing).

**Q1. Evaluate the use of Encryption as a primary security measure for a company. (8 marks)**
- **1. Data Protection (+1)**: Even if a hacker manages to steal the hard drive or intercept the data, it is completely useless without the key.
- **2. Privacy (+1)**: Ensures that only the intended recipient can read personal or sensitive information like medical records.
- **3. Compliance (+1)**: Many laws (like GDPR) legally require companies to encrypt customer data to avoid massive fines.
- **4. Reputation (+1)**: Customers are more likely to trust a company that advertises high-level encryption for its services.
- **5. Integrity (+1)**: If an encrypted file is tampered with, the decryption process will fail, alerting the user to the breach.
- **6. Performance Impact (-1)**: Constantly encrypting and decrypting large volumes of data requires extra CPU power and can slow down the system.
- **7. Key Management (-1)**: If the decryption key is lost or forgotten, the data is permanently unrecoverable, even for the owner.
- **8. Complexity (-1)**: Setting up asymmetric encryption across a large workforce is difficult and requires expert IT staff.
- **9. Backdoor Risks (-1)**: Some governments demand "backdoors" into encryption, which could be discovered and used by criminals.

**Q2. Discuss how a company can defend itself against Social Engineering attacks. (8 marks)**

- **2. Simulated Attacks (+1)**: Sending "fake" phishing emails from the IT department to see who clicks, then providing extra training to those people.
- **3. Strict Policies (+1)**: Having a clear rule that "IT will never ask for your password" prevents employees from falling for pretexting.
- **4. Physical Barriers (+1)**: Using turnstiles or ID card scanners prevents tailgating into sensitive areas.
- **5. Verification Systems (+1)**: Requiring staff to call a known internal number to verify any unusual request before sharing data.
- **6. AUP (Acceptable Use Policy) (+1)**: Making staff sign a document outlining their security responsibilities as part of their contract.
- **7. Regular Audits (+1)**: Checking that server rooms are locked and that employees are not leaving passwords on post-it notes.
- **8. MFA (+1)**: Even if a social engineer steals a password, they cannot log in without the physical token or biometric factor.
- **9. Reporting Culture (+1)**: Encouraging staff to report suspicious behavior immediately without fear of getting in trouble.
- **10. Technical Filters (+1)**: Using advanced email filters to block known phishing links before they even reach the employee's inbox.

- **Hacking**: Use the term **'Unauthorized access'**.
- **Social Engineering**: State that it targets **Human Psychology** rather than computer code.
- **Firewall**: It is a **Filter**, not an antivirus. It monitors **Traffic**, not files.
- **Authentication**: Remember the three factors: **Know, Have, Are**.
- **Pharming**: Different from phishing! Pharming involves **DNS poisoning** to redirect you automatically.

# Chapter 7: The Impact of the Internet 

The internet is a global network of interconnected computers that has fundamentally altered the fabric of human society. It impact spans communication, education, commerce, and social hierarchy.

## The Evolution of Communication Methods
The internet has moved communication from slow, physical, and synchronous to fast, digital, and asynchronous.

### 1. Electronic Mail (Email)
- **Concept**: A digital method for exchanging messages across the internet.
- **Key Features**:
    - **Attachments**: Sending documents, photos, or spreadsheets.
    - **Cc (Carbon Copy)**: Sending a copy to someone for info.
    - **Bcc (Blind Carbon Copy)**: Sending a copy without the main recipient knowing.
    - **Signatures**: Automatic contact info at the bottom.
    - **Folders/Labels**: Organizing thousands of messages.
    - **Auto-responders**: Sending "Out of Office" messages.
- **Comparison to Physical Mail**:
| Feature | Email | Physical Mail (Snail Mail) |
| :--- | :--- | :--- |
| **Speed** | Instant global delivery. | Takes days or weeks. |
| **Cost** | Free (with an internet connection). | Requires stamps and envelopes. |
| **Media** | Links, videos, and dynamic files. | Paper and physical objects only. |
| **Security** | Encrypted but prone to hacking. | Physical tampering is noticeable. |
| **Volume** | Thousands of messages stored digitally. | Bulky and requires physical storage. |

### 2. Instant Messaging (IM)
- **Concept**: Real-time text-based communication between individuals or groups.
- **Impact**:
    - Replaced short phone calls for quick questions.
    - Introduced "Presence" (knowing when someone is active).
    - Introduced "Read Receipts" (knowing when someone saw the message).
    - Heavy use of **Emojis** and **GIFs** to express emotion.
- **Primary Use Cases**:
    - Personal chats (WhatsApp, Messenger).
    - Professional collaboration (Slack, Teams).
    - Customer support chat bubbles on websites.

### 3. Voice over Internet Protocol (VoIP)
- **Concept**: Converting voice into data packets to be sent over the internet instead of traditional phone lines.
- **Benefits**:
    - Much cheaper (often free) for international calls.
    - Can be used on any internet-connected device (Phone, PC, Tablet).
    - Integrates voice with video and data sharing.
- **Drawbacks**:
    - Requires high-speed stable internet.
    - If the internet goes down, the phone doesn't work.
    - Quality can be affected by latency (lag).

### 4. Video Conferencing
- **Concept**: Real-time video and audio communication between two or more people.
- **Requirements**:
    - **Hardware**: High-def camera, noise-canceling microphone, high-res monitor.
    - **Network**: High upload and download bandwidth (min 2Mbps for HD).
    - **Software**: Platforms like Zoom, Microsoft Teams, or Google Meet.
- **Features**:
    - **Screen Sharing**: Presenting documents to the group.
    - **Virtual Backgrounds**: Hiding the user's messy room.
    - **Recording**: Saving the meeting for those who missed it.
    - **Breakout Rooms**: Dividing a large group into small workshops.

## The Impact on Socialising and Community
Traditional communities bounded by geography have been replaced by digital communities bounded by interest.

### Social Networking Sites (SNS)
- **Features of SNS**:
    - **User Profiles**: Digital representations of the individual.
    - **News Feeds**: Algorithmically sorted content from friends.
    - **Like/Share/Comment**: Interactive metrics for engagement.
    - **Groups/Pages**: Centering interactions around specific hobbies or brands.
- **Positive Impacts**:
    - **Global Connection**: Maintaining long-distance family relationships.
    - **Rapid News**: Information on emergencies or events spreads in seconds.
    - **Economic Opportunities**: Small creators can reach a global audience.
- **Negative Impacts**:
    - **Cyberbullying**: Targeted harassment using digital anonymity.
    - **Addiction**: The "validation loop" of likes and notifications.
    - **Privacy Erosion**: Personal data being harvested by companies.
    - **Echo Chambers**: Seeing only viewpoints that you already agree with.

### Virtual Worlds and Online Gaming
- **Concept**: 3D digital environments where users interact via **Avatars**.
- **Social Impact**:
    - High immersion leads to strong emotional bonds between users.
    - Allows people with physical disabilities to interact freely.
    - Can lead to "Gamer Rage" or social withdrawal from the physical world.

## Impact on Lifestyle and Daily Activities
Digital services have "disintermediated" many traditional industries.

### 1. Online Shopping (E-commerce)
| Feature | Benefit to Customer | Benefit to Business |
| :--- | :--- | :--- |
| **Availability** | Shop 24/7 from home. | No need for expensive physical store rent. |
| **Comparison** | Instant price comparisons across 100 sites. | Global customer base (billions of people). |
| **Stock** | Access to niche products not sold locally. | Low overheads for warehouse storage. |
| **Personalization** | Tailored recommendations based on history. | Data on exactly what customers want to buy. |

### 2. Online Banking
- **Key Features**:
    - **Instant Transfers**: Moving money between accounts in seconds.
    - **Scheduled Payments**: Automatic bills and subscriptions.
    - **Mobile Deposits**: Scanning checks with a phone camera.
    - **Security**: Fingerprint login and instant fraud alerts.
- **Societal Impact**:
    - Decline of physical bank branches in rural areas.
    - Increased financial inclusion for people with mobile phones but no bank nearby.

### 3. Entertainment and Media
- **Streaming (Netflix, Spotify, YouTube)**:
    - The "Death of Physical Media" (CDs, DVDs).
    - Immediate access to millions of songs/movies.
    - Subscription models replace buying individual items.
- **Digital Photography**:
    - Unlimited photos at zero cost per shot.
    - Instant editing and global sharing.

### 4. Travel and Tourism
- **Impact**:
    - Online booking (Expedia, Airbnb) has mostly replaced travel agents.
    - In-depth reviews from other travelers help make better choices.
    - Electronic tickets and boarding passes on smartphones.

### 5. Education (E-learning)
- **VLE (Virtual Learning Environment)**: (e.g., Moodle, Google Classroom).
- **MOOCs**: (Massive Open Online Courses) providing university education for free.
- **Benefits**:
    - Learn at your own pace.
    - Access to global experts.
    - Flexible schedule for working adults.

## The Impact on Working Patterns
Technology has enabled a radical shift in *where* and *when* people work.

### 1. Teleworking (Work from Home)
| Stakeholder | Advantage | Disadvantage |
| :--- | :--- | :--- |
| **Employee** | Zero commute time (saves money/stress). | Social isolation and loneliness. |
| **Employee** | Better work-life balance (childcare). | Hard to "switch off" after hours (burnout). |
| **Employer** | Reduced rent and utility costs for offices. | Harder to monitor staff performance. |
| **Employer** | Access to a global talent pool. | Difficulty building a unified company culture. |
| **Society** | Reduced traffic congestion and CO2. | Decline of city-center shops and cafes. |

### 2. Mobile Working
- Working while "on the move" (trains, planes, cafes).
- Enabled by high-speed 4G/5G and portable laptops.
- Allows field workers (engineers, sales) to update plans in real-time.

### 3. Flexible Working
- Choosing your own hours (e.g., 7am - 3pm) instead of a fixed 9-to-5.
- Enabled by cloud collaboration tools where you can pick up work at any time.

### 4. The Gig Economy
- Short-term, freelance contracts managed via apps (Uber, Deliveroo, Upwork).
- **Pros**: Ultimate flexibility for workers; cheap services for consumers.
- **Cons**: No sick pay, no pension, and unstable income (job insecurity).

## The Digital Divide
The gap between the **Information Rich** and the **Information Poor**.

### Causes of the Divide
1.  **Economic**: High cost of devices (MacBooks, High-end PCs) and monthly data plans.
2.  **Geographic**: Rural or mountainous areas lacking fiber-optic or 5G coverage.
3.  **Educational**: Lack of digital literacy or ICT training (common in elderly populations).
4.  **Disability**: Software or hardware not designed for people with specific accessibility needs.
5.  **Political**: Governments blocking or censoring the internet in certain countries.

### Solutions to Bridge the Divide
- **Community Access**: Free Wi-Fi and computers in public libraries and village halls.
- **Government Subsidies**: Grants for telecom companies to lay cables in "unprofitable" rural areas.
- **Mobile Leapfrogging**: Developing 4G/5G mobile internet in areas that never had landlines.
- **Edu-Tech Programs**: Integrating ICT into all school levels to ensure every child is digitally literate.
- **Low-Cost Devices**: Development of $100 tablets or "Lite" versions of popular operating systems.

# Chapter 8: Online Communities 

An online community is a group of people with a shared common interest who communicate, share information, and collaborate via the internet. Each community has a specific **Function** and a set of **Features** that enable that function.

## 1. Social Networking Communities
Forming groups in society is a fundamental human behavior. The internet has provided new ways to form these networks across any distance.
- **Primary Examples**: Facebook, Baidu, Twitter, LinkedIn.
- **Function**: To allow members to connect through shared interests, personal relationships, or professional goals.

### Key Features of Social Networks
- **User Profiles**: 
    - A collection of personal information about a user.
    - Biographical details: Name, gender, DOB, location, language.
    - Personalization: Background images, color schemes, profile photos.
    - Privacy settings: Determining who can see what data (Public vs Private).
- **Connections (Friend/Follow/Connect)**:
    - Features to add people to your primary network.
    - Mutual connections: Seeing who you both know.
- **Stream / Wall / Timeline**:
    - A central place where posts from your network appear automatically.
    - Changed communication by bringing info to the user rather than making them search for it.
- **Status Updates and Posts**:
    - Sending text, images, or links to your network or the public.
- **Targeted Marketing**:
    - Social networks store user info in databases and analyze it.
    - Information is sold to advertisers to show "Relevant Sponsored Posts."
    - This is the primary revenue model for free platforms (e.g., Facebook, Instagram).
- **Groups and Circles**:
    - Dividing your network into sub-groups (e.g., Family, Work, Hobbies).
    - Prevents sending personal info to professional contacts.
- **Tags**:
    - Labels added to content to categorize it.
    - Allows other users to search for specific topics easily.
- **Reactions and Ratings**:
    - Likes, Upvotes, Downvotes, and Emojis.
    - Helps recommend high-quality content to others.
- **Share / Repost**:
    - Circulating content beyond the original creator's network.
    - When shared widely, a post is described as going **'Viral'**.
- **Third-party Integration**:
    - "Login with Facebook" or "Share to Twitter" buttons on other websites.
- **Analytics**:
    - Showing users how many people viewed or reacted to their posts.
    - Helps businesses measure the effectiveness of their marketing.

## 2. Online Gaming Communities
- **Concept**: Platforms where large numbers of people play multiplayer games together.
- **Examples**: PlayStation Network (PSN), Xbox Live, Steam, MMORPGs (World of Warcraft).
- **Function**: To allow members to compete or collaborate in real-time gaming environments.
- **Features**:
    - **Experience Points (XP)**: Tracking and displaying a user's progress and skill level.
    - **In-game Chat**: Voice or text communication during gameplay.
    - **Discussion Forums**: Talking about tactics, walk-throughs, and hidden secrets.
    - **Status Indicators**: Seeing if friends are currently online or "In-game."
    - **Notifications**: Alerts about game events, updates, or friend requests.
    - **Matchmaking**: Finding other players of similar skill levels automatically.

# Chapter 9: The Implications of Digital Technologies 

This chapter explores the legal, ethical, and environmental consequences of our massive reliance on digital devices. Technology does not exist in a vacuum; its use is governed by law and social responsibility.

## Data Protection Legislation
Governments create laws to ensure that organisations handle our personal information responsibly.

### 1. The UK Data Protection Act (DPA)
The DPA establishes the "Data Protection Principles" that every data controller (organisation) must follow.
- **Principle 1**: Data must be used **Fairly and Lawfully**.
- **Principle 2**: Data must be used for **Limited, Specifically Stated Purposes** only.
- **Principle 3**: Data must be **Adequate, Relevant, and Not Excessive** for the task.
- **Principle 4**: Data must be **Accurate** (and kept up to date).
- **Principle 5**: Data must be kept for **No Longer Than Absolutely Necessary**.
- **Principle 6**: Data must be handled according to **People’s Data Protection Rights**.
- **Principle 7**: Data must be kept **Safe and Secure** (encryption, firewalls).
- **Principle 8**: Data must **Not be Transferred Outside the EEA** (European Economic Area) without adequate protection.

### 2. Individual Rights under the DPA
- **Right to Access**: You can request a "Subject Access Request" to see all data held about you.
- **Right to Object**: You can stop an organisation from processing data that causes you distress.
- **Right to Prevent Marketing**: You can "opt-out" of newsletters and sales calls.
- **Right to Automated Decisions**: You can object to a computer making a decision about you (e.g., a loan rejection) without human review.
- **Right to Rectification**: You can force an organisation to fix incorrect data.
- **Right to Compensation**: If a law breach causes you financial or emotional loss, you can sue.

## Copyright and Intellectual Property
Copyright is the legal power that gives creators sole control over how their original work is used.

### Types of Protected Work
- **Software**: Code, games, apps.
- **Media**: Music, movies, books, podcasts.
- **Visuals**: Digital images, photographs, architectural plans.

### Software and Media Licensing
| Licence Type | Description | Flexibility |
| :--- | :--- | :--- |
| **Proprietary** | Paid-for license (e.g., Windows). | Cannot see or edit source code. |
| **Open Source** | Free to use and edit (e.g., Linux). | High - anyone can modify the code. |
| **Creative Commons** | Semi-free for specific uses (e.g., CC-BY). | Varies - may require attribution or no-commercial. |
| **Individual** | For a single person/device. | Low - cannot be shared. |
| **Site License** | For a whole building (e.g., a school). | High - many users on one site. |

### Enforcing Copyright
- **DRM (Digital Rights Management)**: Software locks that prevent copying or sharing a file.
- **Geoblocking**: Restricting content based on a user's location (IP address).
- **Copy Protection**: Physical or digital blocks on hardware to prevent ripping.
- **Lawsuits**: Sued for "Copyright Infringement" (piracy).

## Monitoring Individuals
Digital technology allows governments and companies to track our movements and communications.

### Methods of Monitoring
- **CCTV**: Security cameras. Modern versions use **Facial Recognition**.
- **ANPR**: Automatic Number Plate Recognition for tracking cars.
- **IP/MAC Addresses**: Identifying a specific device every time it connects to a network.
- **GPS Tracking**: Shared via apps (Find My Friends) or hidden trackers.
- **ID/Travel Cards**: Logging every time you swipe through a turnstile or border.
- **Keyloggers**: Recording every keystroke a person types.

### The Privacy Debate
| Benefit | Drawback / Controversy |
| :--- | :--- |
| **Finding lost children** via GPS trackers. | **Erosion of privacy** - feeling "always watched." |
| **Identifying criminals** at sporting events. | **False Positives** in facial recognition (wrongly accused). |
| **Keeping travelers safe** at airports. | **Lack of Trust** - citizens feel like suspects. |
| **Locating friends** for social meetups. | **Stalking risks** if data is leaked or hacked. |
| **Reducing fraud** in bank transactions. | **Energy consumption** of massive server monitoring. |

## Sustainability and the Environment
The manufacturing and disposal of ICT equipment has a massive global ecological footprint.

### Environmental Problems
- **Energy Use**: Digital devices and data centres consume massive amounts of non-renewable power (coal/gas).
- **Cooling**: Data centres generate huge heat; cooling systems use millions of litres of fluid and massive power.
- **Poisonous Substances**: Devices contain **Bromine, Mercury, and Chlorine** which are toxic.
- **E-waste**: Throwing away old phones leads to toxins leaking into soil and water (ecology).

### Mitigation (Solutions)
- **Renewable Energy**: Using solar, wind, or hydroelectric power for data centres.
- **Cold Climate Building**: Building data centres in the Arctic (e.g., Sweden) to use the natural air for cooling.
- **Recycling Schemes**: Formal programs to extract gold and rare metals from old gadgets safely.
- **Legislation**: Laws like the EU **WEEE Directive** (Waste Electrical and Electronic Equipment).
- **Energy Efficient Hardware**: Designing CPUs that use less "Idle" power.

## Health and Safety in ICT
Using technology incorrectly can cause long-term physical and mental harm.

| Risk | Primary Cause | Mitigation (Prevention) |
| :--- | :--- | :--- |
| **Eye Fatigue / Dryness** | Staring at screens / Blue light. | Take breaks, use blue-light filters, reduce glare. |
| **RSI (Repetitive Strain)** | Typing/Clicking for hours. | Use ergonomic keyboards, wrist pads, and mouse support. |
| **Back/Neck Ache** | Poor posture (slouching). | Adjustable chairs, monitor at eye level, feet flat. |
| **Trip Hazards** | Trailing power/ethernet cables. | Use trunking, tidy cables, avoid "cable spaghetti." |
| **Electric Shock** | Damaged cables or wet hands. | Regular cable checks, use of **RCDs**, no liquids. |
| **Fire** | Overloaded sockets / Overheating. | Don't daisy-chain plugs, ensure proper fan airflow. |
| **Injury / Death** | Distraction while walking/driving. | Do not use phones near roads or machinery. |

## Ethics and Artificial Intelligence (AI)
As computers make more decisions, new ethical questions arise.
- **Self-Driving Cars**: Should the car hit a dog or swerve and hit a person? (The "Trolley Problem").
- **Judgment vs Logic**: Computers lack "Common Sense" - they follow rules exactly even if the result is absurd.
- **Legal Responsibility**: If an AI causes an accident, is it the coder, the car owner, or the software company's fault?
- **Bias**: AI trained on biased human data can make racist or sexist decisions in hiring or policing.

# Chapter 10: Online Information 

Information is the lifeblood of the digital age. Moving from raw **Data** to meaningful **Information** requires context, organization, and evaluation. This chapter covers how to find, evaluate, and use online information responsibly.

## 1. Information Sources
Information can be classified by where it originated and who created it.

### Primary vs Secondary Sources
| Source Type | Definition | Examples |
| :--- | :--- | :--- |
| **Primary Source** | Data or media created by you first-hand. | Your own photos, interviews you conducted, your logs. |
| **Secondary Source** | Information created by someone else that you use. | Books, newspapers, websites, documentaries, maps. |

### The "Fitness for Purpose" Test
Information must be "Fit for Purpose," meaning it is suitable for the specific task you are doing. Evaluation criteria include:
- **Accuracy**: Is the information correct and factually true?
- **Age**: Is the information up-to-date (e.g., current stock prices) or historically relevant?
- **Reliability**: Can you trust the provider? (e.g., a university vs an anonymous blog).
- **Bias**: Is the information one-sided or intended to influence you?
- **Relevance**: Does it actually help solve your specific problem?

## 2. Using Search Engines Effectively
A search engine is a service that uses a database of web page indexes to help users find information.

### How a Search Engine Works
1.  **Crawling**: Software "spiders" visit millions of websites and follow every link.
2.  **Indexing**: The engine stores the words and location of every page in a massive database.
3.  **Searching**: When you type keywords, the engine searches its own database (not the live web).
4.  **Ranking**: Algorithms determine which pages are most relevant and show them first.

### Search Engine Features
- **Keywords**: The specific terms used for the search. Simple is usually better.
- **Search Types**: Filtering by Media Type (Images, Videos, Maps, Shopping, News).
- **Search Tools (Filters)**:
    - **Size**: For images (Large, Medium, Icon).
    - **Color**: Finding images with specific dominant colors.
    - **Time**: (Past hour, 24 hours, year) - critical for news.
    - **Usage Rights**: Finding images that are free to reuse (Creative Commons).
- **Autofill**: The engine suggests common phrases as you type based on popular searches.
- **Suggested Sites**: Based on your personal **Browsing History** stored in cookies.

### Advanced Search Syntax (Boolean Operators)
| Operator | Logic | Result |
| :--- | :--- | :--- |
| **AND (+)** | Both words must be present. | `cat + dog` (finds pages with both). |
| **NOT (-)** | Exclude a specific word. | `jaguar -car` (finds the animal, not the vehicle). |
| **Phrase ("")** | Exact word order. | `"to be or not to be"` (finds that exact quote). |
| **OR** | Either word can be present. | `college OR university`. |
| **Asterisk (*)** | Wildcard for unknown words. | `the * of the rings`. |

## 3. Information Reliability and Integrity
Just because information is easy to find on the internet does not mean it is true.

### Indicators of Reliability
1.  **URL Ending**: `.gov` (Government), `.edu` (Education), and `.ac.uk` are generally more trustworthy than `.com` or `.org`.
2.  **Author Credentials**: Does the writer have a degree or professional experience in the topic?
3.  **Citations**: Does the article link to its primary sources?
4.  **Professionalism**: Are there spelling mistakes, broken links, or aggressive "clickbait" ads?
5.  **Cross-Referencing**: Does the same fact appear in three other independent, trustworthy sources?

### Detecting Bias
- **Funding**: Who paid for the research? (e.g., a study on sugar funded by a soda company).
- **Language**: Does it use "Loaded words" intended to trigger an emotional response?
- **Omission**: Does it leave out facts that would weaken its argument?
- **Perspective**: Is it providing a balanced view of both sides of a debate?

## 4. Responsible Use of Information
Using information incorrectly can lead to legal or academic penalties.

### Plagiarism vs Paraphrasing
- **Plagiarism**: Copying someone else's work and presenting it as your own.
- **Paraphrasing**: Rewriting information in your own words while still giving credit.
- **Why Paraphrase?**: It shows you actually **Understand** the topic rather than just copying pixels.

### Acknowledging Sources (Citations)
When using information online, you should include:
1.  **URL**: The full web address.
2.  **Author**: If known.
3.  **Date Accessed**: Because web content can change or disappear.
4.  **Title**: The name of the article or page.

### Understanding Cookies
Cookies are small data files placed on your device to facilitate these services.

| Type | Duration | Usage Case |
| :--- | :--- | :--- |

# Chapter 12: The Cloud 

"The Cloud" is a metaphor for the internet, specifically when it is used to provide on-demand processing power (Computing) or data storage, rather than relying on local hardware.

## 1. Cloud Computing vs Local Computing
Where a program "runs" determines its performance, features, and accessibility.

### Local Computing (Installed Software)
- **Concept**: The program files are stored on the device's hard drive (HDD/SSD) and the processing is done by the local CPU/RAM.
- **Benefits**:
    - **Performance**: Zero latency; no network delay when clicking a button.
    - **Offline Use**: Works perfectly without an internet connection.
    - **Feature Rich**: Can handle massive tasks like 4K video editing or advanced 3D modeling.
- **Drawbacks**:
    - **Fixed to Hardware**: Only works on the device where it is installed.
    - **Maintenance**: User must manually download and install updates and patches.
    - **Storage**: Takes up significant space on the local drive.

### Cloud Computing (Hosted Applications / SaaS)
- **Concept**: The application runs on a remote server; only the user interface is sent to the client's browser.
- **Benefits**:
    - **Accessibility**: Log in from any device (phone, laptop, tablet) and see the same work.
    - **Collaboration**: Multiple people can edit the same document simultaneously (Real-time).
    - **Automatic Updates**: The provider updates the software on the server; the user always has the latest version.
- **Drawbacks**:
    - **Internet Dependent**: Completely useless if the Wi-Fi or 4G drops.
    - **Latency**: High-bandwidth tasks may feel "laggy" or slow to respond.
    - **Simplified Features**: Often lacks the "Pro" features of desktop software to keep bandwidth low.

## 2. Infrastructure as a Service (IaaS)
The cloud isn't just for software; it's used for fundamental computing infrastructure.

- **Virtual Servers**: Companies can "Rent" a computer that doesn't physically exist in their office.
- **Processing Power (CPU Cycles)**: Massive mathematical or statistical jobs can be offloaded to thousands of cloud CPUs for completion in hours instead of days.
- **Remote Networking**: Setting up firewalls and routers in the cloud to connect remote offices.

## 3. Online Data Storage
Storing digital assets in data centers rather than on local disks.

### Benefits of Cloud Storage
| Feature | Explanation |
| :--- | :--- |
| **Automatic Backups** | The system can mirror your files the second you save them, protecting against fire or theft of your physical device. |
| **Global Access** | Access your holiday photos or work documents from any country with an internet connection. |
| **Unlimited Scale** | You don't need to buy a bigger hard drive; you simply pay a few extra pounds for 1TB or 10TB of space. |
| **Version History** | Cloud providers often keep previous versions of files, allowing you to "Undo" a mistake from 3 days ago. |
| **Collaboration** | Sharing a 2GB video file is done by sending a link, which is faster and safer than using a fragile USB stick. |

### Risks and Drawbacks
| Risk | Description |
| :--- | :--- |
| **Data Protection** | Storing files in another country may subject them to foreign laws that provide less privacy protection. |
| **Security Breach** | A massive cloud provider is a "Honey Pot" target for high-level global hackers. |
| **Transfer Speed** | Uploading or downloading 10GB of video is much slower over Wi-Fi than over a local cable connection. |
| **Provider Reliance** | If the cloud company goes bankrupt, your data could potentially disappear forever. |

## 4. Societal Impact: The Digital Divide
The cloud has been a major force in making technology more accessible to everyone.

- **Chromebooks and Netbooks**:
    - These are "Cloud-First" devices that are often extremely cheap (often under £200).
    - Because they run everything in the cloud, they don't need expensive internal parts like massive hard drives or high-tier CPUs.
    - **Impact**: Allows schools and students in lower-income regions to access world-class software at a fraction of the cost.
- **Mobility and the "Digital Nomad"**:
    - The cloud allows people to work from cafés, home, or while traveling, improving work-life balance for many and reducing the need for physical urban traffic.
- **Education Access**:
    - Students in remote areas can access MOOCs (Massive Open Online Courses) from elite universities via the cloud for free or low cost.

## 5. Impact on Organisations
The cloud is a "Disruptive Technology" that has forced businesses to rethink their entire IT strategy.

### Business Transformation Table
| Area | Impact | Detailed Benefit |
| :--- | :--- | :--- |
| **Cost Management** | CAPEX to OPEX | Instead of buying expensive servers up-front (Capital Expenditure), firms pay a monthly sub (Operating Expenditure). |
| **IT Staffing** | Outsourced Expertise | The cloud provider handles security patches and hardware repairs, so the firm needs fewer IT staff for basic maintenance. |
| **Disaster Recovery** | Built-in Resilience | If an office burns down, the business data is safe in the cloud and can be accessed from a new location instantly. |
| **Scalable Growth** | Instant Deployment | A startup can go from 10 users to 10 million users without having to build their own physical data center. |
| **Competitive Edge** | Software Access | Small firms can now use the same high-end software (CRM, ERP) as massive corporations for a low entry price. |

## 6. Cloud-Based Communities
The cloud is the engine behind modern social interaction and knowledge sharing.

- **Social Networking**: Sites like Facebook and LinkedIn are essentially massive cloud-based databases of human relationships.
- **Online Workspaces**: Slack, Microsoft Teams, and Trello allow teams to organize tasks and communicate in a persistent, searchable cloud environment.
- **User-Generated Content**: YouTube, TikTok, and Instagram are cloud storage platforms that allow for the global sharing of video and image media.
- **Collaborative Knowledge**: Wikis and forums allow global populations to build reference databases (like Wikipedia) together in real-time.

# Chapter 13: Applying ICT 

Applying ICT is the process of selecting the right digital tools to solve a problem and produce a product that is "Fit for Purpose" and appropriate for its "Target Audience."

## 1. Data vs Information
The fundamental transition in ICT is moving from meaningless raw values to structured knowledge.
- **Data**: Raw, unprocessed numbers, characters, images, or sounds.
    - Example: `25`. (Is it a temperature? An age? A speed?)
- **Information**: Data that has been processed and given **Context**.
    - Example: `25.0 °C` (processed as a temperature measurement for a weather report).
- **The Process**: Data + Processing + Context = Information.

## 2. Selecting the Best Software
Using the correct tool for a task ensures efficiency and professional results.

| Task Category | Recommended Software | Why? |
| :--- | :--- | :--- |
| **Writing and Layout** | Word Processor | Features for paragraphs, spell-check, and mail merge. |
| **Numeric Modeling** | Spreadsheet | Can perform complex math and create charts instantly. |
| **Record Management** | Database | Can handle millions of rows with fast searching/filtering. |
| **Visual Creation** | Graphics Editor | Allows for layers, transparency, and pixel-level editing. |
| **Interactive Info** | Web Authoring | Creates hyperlinked pages accessible via a browser. |
| **Public Speaking** | Presentation | Formats content into slides with transitions and timing. |

## 3. The Lifecycle of a Digital Product
1.  **Enter**: Inputting raw data (typing text, uploading photos).
2.  **Organise**: Arranging content logically (Bullet points, tables, slide order).
3.  **Develop**: Adding value (formulas in spreadsheets, links in web pages).
4.  **Refine**: Polishing and checking (Thesaurus, spell-checker, absolute cell references).
5.  **Format**: Making it "look the part" (Fonts, colors, reports, master slides).

## 4. Design Standards and House Styles
Organisations use strict rules to maintain their **Corporate Identity** (Branding).
- **House Style**: A document that specifies:
    - **Logo Position**: e.g., "Top right, must have 2cm of white space around it."
    - **Typeface**: Specific fonts used (e.g., Helvetica for headings).
    - **Colour Palette**: Exact hex codes for brand colors.
    - **Templates**: Pre-designed layouts for letters, memos, or faxes.
- **Benefits of House Style**:
    - Ensures professionalism across thousands of physical and digital products.
    - Makes the brand instantly recognizable to customers.
    - Saves time for designers by removing "guesswork."

## 5. Typography and Colour Theory
- **Serif Fonts**: (e.g., Times New Roman). Have small "feet" on letters. Better for **Printed** books as they guide the eye.
- **Sans Serif Fonts**: (e.g., Arial, Verdana). No feet. Better for **Digital Screens** (especially low-res) as they look cleaner and are more accessible.
- **White Space**: The empty area on a page.
    - Too much: Makes the product look empty and unfinished.
    - Too little: Makes the product look crowded and stressful.
- **Colour Emotion**:
    - **Red**: Excitement, Danger, Urgency.
    - **Blue**: Professionalism, Trust, Calm.
    - **Yellow**: Fun, Happiness, Warning.
- **Contrast**: Ensuring text is easy to read against a background (e.g., Black text on white).

# Chapter 14: File Management 

Effective file management is critical for productivity and data safety. Without organization, files become "lost" on high-capacity drives, and work can be overwritten by accident.

## 1. Naming Conventions
A good filename tells the user exactly what is inside without them having to open the file.

### Sensible Filename Rules
- **Be Descriptive**: `Sports_Club_Poster.pdf` is better than `document1.pdf`.
- **Use Snake Case**: Use underscores (`_`) instead of spaces. 
    - *Why?* Some older software and servers treat spaces as "Illegal characters," which can break web links.
- **Reverse Date Order (YYYYMMDD)**: 
    - Example: `20241225_Invoice.doc`.
    - *Why?* If you have 100 invoices, sorting them by "Name" will also sort them perfectly by "Date."
- **Avoid Special Characters**: Don't use `?`, `*`, `/`, or `\`. These are reserved by the Operating System for path and command logic.

### Version Control (Versioning)
When a file is updated, you should save it as a new version rather than overwriting the original.
- **Simple Versioning**: `Assignment_v1.doc`, `Assignment_v2.doc`.
- **Pro Versioning (Build numbers)**: `Project_v1.3.20241012.05`.
    - **1**: Major version (Big changes).
    - **3**: Revision (Minor fixes).
    - **20241012**: The build date.
    - **05**: The sequence number for that day.

## 2. Hierarchical Folder Structures
Folders should be nested in a way that makes logical sense, moving from **General** to **Specific**.
- **Top Level**: `School_Work`
- **Sub-Level**: `ICT`, `History`, `Science`
- **Topic Level**: `ICT/Presentations`, `ICT/Spreadsheets`
- **Asset Level**: `ICT/Presentations/Images`

## 3. File Formats and Extensions
The file extension (the 3 or 4 letters after the dot) tells the OS which application to use.
- **Text**: `.txt` (Plain), `.rtf` (Rich Text), `.docx` (Word).
- **Images**:
    - `.jpg`: Good for photos (Lossy compression).
    - `.png`: Good for logos (Lossless, supports transparency).
    - `.gif`: Animaged or simple colors.
    - `.svg`: Vector graphic (Scalable).
- **Audio**: `.mp3` (Compressed), `.wav` (Uncompressed, high quality).
- **Video**: `.mp4`, `.mov`.
- **Archive**: `.zip` (Many files compressed into one).

## 4. File Security and Backups
- **Password Protection**: Many apps (Word/Excel) allow you to "Encrypt with Password" so unauthorized users can't read the file.
- **Read-Only Mode**: Allowing people to see the file but not change it.
- **The "3-2-1" Backup Rule**:
    - **3** copies of your data.
    - **2** different local media (e.g., PC + External Drive).
    - **1** copy off-site (e.g., The Cloud).

# Chapter 15: Word Processing 

Word processing software is designed for the creation, editing, and formatting of text-based documents. It is the primary tool for producing business letters, academic reports, and promotional flyers.

## 1. Entering and Editing Content
- **Insertion Point (Cursor)**: The flashing vertical line that indicates where the next character typed will appear.
- **Word Wrap**: The software automatically moves text to a new line when it reaches the right margin; the user only presses "Return" for a new paragraph.
- **Key Editing Tools**:
    - **Cut/Copy/Paste**: Using the "Clipboard" to move or duplicate text and images.
    - **Undo/Redo**: Reversing mistakes or re-applying changes instantly.
    - **Find and Replace**: Searching for every instance of a word and changing it (e.g., changing "Chairman" to "Chairperson" throughout a 100-page report).

## 2. Document Formatting
Formatting changes the visual appearance and hierarchy of the information.

### Font and Character Effects
- **Typeface (Font)**: Choosing between Serif and Sans Serif styles for readability.
- **Size**: Measured in points (pt). Headings are usually 14-18pt, body text 10-12pt.
- **Styles**:
    - **Bold**: For emphasis and headings.
    - **Italic**: For book titles, foreign words, or subtle emphasis.
    - **Underline**: For links or older report styles (less common now).
- **Subscript/Superscript**: For chemical formulas (H₂O) or mathematical powers (10²).

### Paragraph and Page Layout
- **Alignment**:
    - **Left**: Standard for most Western documents.
    - **Center**: Used for titles and invitations.
    - **Right**: Used for dates and addresses in formal letters.
    - **Justified**: Text is flush on both margins; used in newspapers and books for a formal, clean look.
- **Line Spacing**: The vertical gap between lines (Single, 1.5, Double).
- **Margins**: The "White space" around the edge of the page.
- **Orientation**: **Portrait** (Vertical) vs **Landscape** (Horizontal).
- **Headers and Footers**: Text that repeats on every page (e.g., Page numbers, Document title, Company name).

## 3. Integrating Digital Assets
Modern word processors allow for the combination of many types of data.
- **Tables**: Organising data into rows and columns.
    - **Merge Cells**: Combining multiple cells into one (for a top header).
    - **Borders and Shading**: Using lines and background colors to distinguish data.
- **Images and Graphics**:
    - **Text Wrapping**: Deciding how text flows around an image (Square, Tight, Behind Text, In Front of Text).
    - **Resizing/Cropping**: Changing the dimensions or removing unwanted outer parts of a photo.
- **Group/Layering**: Treating multiple shapes as one object or moving images "Behind" or "In front" of each other.

## 4. Mail Merge: Automating Documentation
Mail Merge is the process of combining a **Standard Document** with a **Data Source**.
- **The Document**: Contains fixed text and **Merge Fields** (placeholders like `<<First_Name>>`).
- **The Data Source**: A database or spreadsheet containing the unique data for hundreds of people (Name, Address, Balance).
- **The Result**: A set of personalized documents (one for each person in the database).
- **Benefits**:
    - Massive time savings when sending thousands of letters.
    - Ensures consistency and reduces the chance of manual typing errors for names/addresses.

# Chapter 16: Graphics 

Graphics software is used to create, manipulate, and export visual content for use in websites, brochures, and digital presentations.

## 1. Bitmap vs Vector Graphics
The two primary ways computers store and render images.

### Bitmap (Raster) Graphics
- **Concept**: An image made of a grid of thousands or millions of tiny squares called **Pixels**.
- **Key Characteristics**:
    - **Resolution Dependent**: If you enlarge the image, it will **Pixelate** (look blocky).
    - **High Detail**: Best for complex photographs where every pixel has a unique color.
    - **File Size**: Large, as the computer must store data for every single pixel.
- **Common Formats**: `.jpg`, `.png`, `.gif`, `.tiff`.

### Vector Graphics
- **Concept**: An image stored as a series of mathematical equations and co-ordinates.
- **Key Characteristics**:
    - **Resolution Independent**: Can be scaled to the size of a skyscraper or a postage stamp without losing any quality (No pixelation).
    - **Stylised Look**: Best for logos, icons, cartoons, and technical diagrams.
    - **File Size**: Very small, as it only stores the math for the lines and circles.
- **Common Formats**: `.svg`, `.ai`, `.eps`.

## 2. Image Attributes and Quality
- **Resolution**: Measured in PPI (Pixels Per Inch) or DPI (Dots Per Inch). 72 DPI is standard for web; 300 DPI is standard for high-quality print.
- **Colour Depth**: The number of bits used to represent the color of a single pixel. Higher depth means more realistic colors (e.g., 24-bit "True Color").
- **Aspect Ratio**: The proportional relationship between an image's width and height (e.g., 16:9 for widescreen video).

## 3. Editing and Manipulation Techniques
- **Cropping**: Removing the outer edges of an image to change its composition or focus.
- **Selection Tools**: 
    - **Marquee**: Selecting rectangular or circular areas.
    - **Lasso**: Freehand selection.
    - **Magic Wand**: Selecting all pixels of a similar color instantly.
- **Layers**: Stacking elements on the **Z-axis**. This allows you to edit the background without affecting the person in the foreground.
- **Opacity**: Changing how transparent a layer is (0% = Invisible, 100% = Solid).
- **Filters/Effects**: 
    - **Brightness/Contrast**: Adjusting light and dark.
    - **Blur/Sharpen**: Making an image softer or clearer.
    - **Cloning**: Using a "Stamp" tool to copy one part of an image over another (e.g., removing a pimple).

## 4. Metadata (EXIF Data)
Hidden information stored inside an image file, including:
- **Date and Time** the photo was taken.
- **Camera Settings**: Aperture, Shutter Speed, ISO.
- **GPS Location**: Where the photo was snapped (Geotagging).
- **Copyright Info**: Who owns the image.

# Chapter 19: Spreadsheets 

Spreadsheets are powerful tools for managing numeric data, performing complex calculations, and modeling "What-if" scenarios.

## 1. Structure and Data Types
- **The Grid**: Composed of **Columns** (Letters A, B, C...) and **Rows** (Numbers 1, 2, 3...).
- **Cell**: The intersection of a row and column (e.g., `B4`).
- **Active Cell**: The cell currently selected, outlined with a thick border.
- **Range**: A group of connected cells, written as `FirstCell:LastCell` (e.g., `A1:C10`).
- **Data Categories**:
    - **Labels (Text)**: Qualitative data, usually aligned to the **Left**.
    - **Values (Numbers)**: Quantitative data, usually aligned to the **Right**.
    - **Formulae**: Instructions that start with an `=` sign to perform a calculation.

## 2. Cell Formatting and Error Handling
- **Number Formats**:
    - **Currency**: Adding `£` or `$` symbols and 2 decimal places.
    - **Percentage**: Converting 0.05 into `5%`.
    - **General**: Default format with no specific rules.
- **Column Management**:
    - **Auto-fit**: Double-clicking the boundary between column headers to fit the widest piece of text.
    - **####### Error**: This occurs when a number is too long for the current column width; it is fixed by widening the column.
- **Merge & Centre**: Combining multiple cells into one large cell and centering the text (common for titles).
- **Text Wrap**: Forcing long text onto multiple lines within a single cell.

## 3. Formulae and Referencing
Formulae allow the spreadsheet to update results automatically if the input values change.

### Relative vs Absolute Referencing
| Ref Type | Example | Behavior when copied |
| :--- | :--- | :--- |
| **Relative** | `A1` | Changes based on the new position (e.g., becomes `B2`). |
| **Absolute** | `$A$1` | Stays fixed to that exact cell no matter where the formula is moved. |

### Logical Functions
- **SUM**: Adds a range of numbers `=SUM(A1:A50)`.
- **AVERAGE**: Calculates the mean.
- **MIN/MAX**: Finds the lowest or highest value in a group.
- **COUNT**: Counts cells containing only **Numbers**.
- **IF**: Performs a logic test: `=IF(B4>50, "Pass", "Fail")`.
- **VLOOKUP**: Searches for a value in a table and returns data from a specific column.

## 4. Data Visualization (Charts)
Charts turn dry numbers into visual stories that are easier for humans to analyze.
- **Pie Chart**: Shows parts of a whole (e.g., Market share).
- **Bar/Column Chart**: Comparing individual items (e.g., Sales per month).
- **Line Graph**: Showing trends over time (e.g., Temperature changes).
- **Scatter Graph**: Showing a relationship or correlation between two variables.

## Past Paper Questions

**Q1. Evaluate the use of 'Absolute Cell Referencing' ($) for a tax calculation spreadsheet. (8 marks)**
- **1. Stability (+1)**: If the "Tax Rate" is in cell `B1`, using `$B$1` ensures every row in a 500-item invoice list uses the same correct rate.
- **2. Error Reduction (+1)**: Prevents the formula from "drifting" into empty cells when dragged down, which would cause `#VALUE!` errors.
- **3. Ease of Update (+1)**: When the government changes the tax rate, the manager only has to update one cell (`B1`) instead of 500.
- **4. Consistency (+1)**: Ensures that the calculation logic is identical for every customer or product in the sheet.
- **5. Audit Trail (+1)**: Makes it easier for an auditor to see that the whole document relies on one official variable.
- **6. Initial Complexity (-1)**: Beginners often forget to add the `$` signs, leading to massive logic errors that are hard to spot.
- **7. Lack of Flexibility (-1)**: If different rows *should* have different rates, absolute referencing will incorrectly apply the same rule to all.
- **8. Slower Creation (-1)**: Manually typing `$` or pressing F4 for every reference takes slightly longer than just clicking.
- **9. Visual Clutter (-1)**: Long, complex formulas filled with `$` signs can be more difficult for a human to read and debug.
- **10. Modeling Benefit (+1)**: Allows for fast "What-if" analysis by changing the absolute cell and seeing the global impact immediately.

**Q2. Discuss the advantages of using a Scatter Graph versus a Bar Chart for a science experiment. (8 marks)**
- **1. Correlation (+1)**: A scatter graph shows if there is a relationship between two variables (e.g., Heat vs Growth).
- **2. Trend Lines (+1)**: You can add a "Line of Best Fit" to a scatter graph to predict future results.
- **3. Outlier Identification (+1)**: A single "bad" result is instantly visible as a dot far away from the others.
- **4. Continuous Data (+1)**: Scatter graphs are better for data that isn't in fixed categories (e.g., Time in seconds).
- **5. Comparison (-1)**: A Bar Chart is better if the data is in discrete categories (e.g., Type of Plant A, B, and C).
- **6. Simplicity (+1)**: Bar charts are much easier for a non-scientific audience to understand at a glance.
- **7. Data Gaps (+1)**: Scatter graphs don't require data for every point; they just plot what exists.
- **8. Precision (-1)**: It can be harder to read the exact value of a tiny dot on a scatter graph compared to a tall bar.
- **9. Multiple Series (+1)**: You can plot two sets of data on one scatter graph and see if they overlap or diverge.
- **10. Labeling (+1)**: Bar charts allow for clear text labels on the X-axis, which scatter graphs often lack.

## Where you can lose marks
- **#######**: Remember this means the **Column is too narrow** for the number.
- **Absolute vs Relative**: Be ready to explain that **Absolute** stays the same when copied.
- **IF Function**: Use the syntax: **(Test, what if True, what if False)**.
- **VLOOKUP**: Know that the first column must contain the search value.
- **Replication**: This is the term for **'dragging down'** a formula to multiple cells.
- **Modeling**: Explain this as **'What-if Analysis'** (changing inputs to see output changes).

# Chapter 20: Database Management 

Databases are structured collections of data that allow for efficient storage, retrieval, and analysis of vast amounts of information.

## 1. Database Terminology and Structure
A database is organized into a hierarchy of components.

### Tables, Records, and Fields
- **Table**: A collection of data about a specific **Entity** (e.g., "Customers" or "Products").
- **Record (Row)**: A single set of data about one specific item in the table (e.g., all data for Customer #402).
- **Field (Column)**: A specific category of information within a record (e.g., "Surname").

### Keys and Relationships
- **Primary Key**: A field that uniquely identifies every record in a table. It cannot be blank, and no two records can have the same value (e.g., StudentID).
- **Foreign Key**: A primary key from one table that appears in another table to link them together.
- **Relational Database**: A database containing multiple tables that are linked by keys to reduce **Data Redundancy** (storing the same data twice).
- **Referential Integrity**: A set of rules that ensures relationships between tables remain consistent. For example, the system will prevent you from deleting a "Customer" if they still have "Orders" linked to them.

## 2. Advanced Data Validation
Validation is an automatic check performed by the computer to ensure data is sensible and follows the rules of the field.

| Check Type | Description | Example |
| :--- | :--- | :--- |
| **Presence Check** | Ensures a field is not left blank. | Mandating a "Postcode" for a delivery. |
| **Range Check** | Ensures a number falls within a set limit. | A "Mark" must be between 0 and 100. |
| **Length Check** | Ensures text has a specific number of characters. | A "Password" must be at least 8 characters. |
| **Type Check** | Ensures the data is the correct category. | Not allowing letters in a "Phone Number" field. |
| **Format Check** | Ensures data follows a specific pattern. | A Date must be DD/MM/YYYY. |
| **Lookup Check** | Users must choose from a predefined list. | Selecting "Male" or "Female" from a dropdown. |

## 3. Retrieving and Outputting Data
- **Queries**: Using "Search Criteria" to filter the database.
    - **Single Criterion**: `City = "London"`.
    - **Multiple Criteria (AND)**: `City = "London" AND Age > 18`.
    - **Logical OR**: `City = "London" OR City = "Paris"`.
- **Forms**: A user-friendly layout for entering or viewing one record at a time, protecting the underlying table structure from accidental changes.
- **Reports**: A professional document used to present query results, often featuring **Grouping** (e.g., grouping sales by region) and **Summaries** (e.g., total sales at the bottom).

## Past Paper Questions

**Q1. Evaluate the use of a 'Relational Database' over a 'Flat-File Database' for a large hospital. (8 marks)**
- **1. Data Redundancy (+1)**: Prevents patient names from being typed 50 times for 50 different appointments; it's stored once and linked.
- **2. Data Consistency (+1)**: If a patient moves house, the address is updated in one table, and it changes everywhere automatically.
- **3. Storage Efficiency (+1)**: Because data isn't repeated, the file size is much smaller, which is vital for millions of hospital records.
- **4. Data Integrity (+1)**: Referential integrity ensures that a doctor can't be assigned to a surgery if they don't exist in the "Staff" table.
- **5. Security (+1)**: Different staff can have different access levels; nurses might see "Medication" but not "Payroll" data.
- **6. Initial Complexity (-1)**: Setting up multiple tables and complex relationships requires a highly skilled "Database Administrator."
- **7. Processing Speed (-1)**: "Joining" three or four tables together for a report can be slower than reading one simple flat file.
- **8. Software Cost (-1)**: Relational systems (like SQL Server or Access) are more expensive to buy and maintain than simple spreadsheets.
- **9. Training Stress (-1)**: Medical staff might find a relational system harder to navigate than a simple list-based flat file.
- **10. Scalability (+1)**: As the hospital grows, a relational database can easily handle billions of rows without breaking.

**Q2. Discuss the importance of 'Validation' versus 'Verification' for an online banking system. (8 marks)**
- **1. Sensibility (Validation) (+1)**: A range check ensures a customer cannot try to withdraw a "Negative" amount of money.
- **2. Formatting (Validation) (+1)**: Ensures a Sort Code is exactly 6 digits, preventing messy data entry before it reaches the server.
- **3. Automated Speed (+1)**: Validation happens instantly as the user types, providing immediate feedback for mistakes.
- **4. Human Error (Verification) (+1)**: Validation won't stop you entering `£100` if you meant `£10`; but "Double entry" verification might.
- **5. Proofreading (Verification) (+1)**: Bank staff must visually check (verify) that the name on a new ID matches the name typed into the system.
- **6. Fraud Prevention (+1)**: Type checks prevent hackers from injecting "Code" into number-only fields (SQL Injection protection).
- **7. Time Consuming (Verification) (-1)**: Typing a password twice (Double entry) is safer but annoys customers who want speed.
- **8. Data Integrity (+1)**: Validation ensures that every record in the bank's database follows the same strict structural rules.
- **9. Business Logic (+1)**: Validation can enforce complex rules, like "Age must be over 18 for a credit card application."
- **10. Total Accuracy (+1)**: A high-security system must use BOTH: Validation to catch structural errors and Verification to catch human typos.

## Where you can lose marks
- **Validation vs Verification**: **Validation** is by the computer (sensibility); **Verification** is by the human (accuracy).
- **Primary Key**: Use the word **'Unique'** to describe it.
- **Foreign Key**: Explain it as the link used to create a **'Relationship'**.
- **Data Redundancy**: This is the term for **'repeating data unnecessarily'** in multiple places.
- **Query Criteria**: Use the terms **'AND'**, **'OR'**, and **'Wildcards'** (like `*`).
- **Reports**: Mention that they are used for **'Professional Printed Output'**.
